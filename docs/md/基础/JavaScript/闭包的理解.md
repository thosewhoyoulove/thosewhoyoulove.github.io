# 闭包的理解

## 面试定位

闭包是 JavaScript 高频题。面试官通常会从“闭包是什么”继续追问到作用域链、变量生命周期、模块封装、循环中的闭包、内存泄漏。

## 核心原理

闭包是指函数能够访问其词法作用域中的变量，即使这个函数在原作用域之外执行。

更具体地说：函数创建时会保存对外层词法环境的引用，只要这个函数还可达，它引用的外层变量就不会被释放。

```javascript
function createCounter() {
    let count = 0;

    return function increment() {
        count++;
        return count;
    };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

`createCounter` 执行结束后，它的执行上下文会出栈，但 `increment` 仍然引用着 `count` 所在的词法环境，所以 `count` 不会被回收。

## 作用域链

JavaScript 使用词法作用域，也就是函数能访问哪些变量由代码书写位置决定，而不是由调用位置决定。

```javascript
const name = "global";

function outer() {
    const name = "outer";

    return function inner() {
        console.log(name);
    };
}

const fn = outer();
fn(); // "outer"
```

`inner` 定义在 `outer` 内部，所以它沿着作用域链找到的是 `outer` 的 `name`。

## 常见应用

### 数据私有化

```javascript
function createStore() {
    let state = {};

    return {
        get(key) {
            return state[key];
        },
        set(key, value) {
            state[key] = value;
        },
    };
}

const store = createStore();
store.set("token", "abc");
console.log(store.get("token")); // "abc"
```

外部不能直接访问 `state`，只能通过暴露的方法读写。

### 函数柯里化

```javascript
function add(a) {
    return function (b) {
        return a + b;
    };
}

const add10 = add(10);
console.log(add10(3)); // 13
```

`add10` 记住了第一次调用传入的 `a`。

### 保存异步上下文

```javascript
function requestWithTrace(traceId) {
    return fetch("/api").then(() => {
        console.log("trace:", traceId);
    });
}
```

异步回调执行时仍然能访问创建它时的 `traceId`。

## 循环中的闭包

经典问题：

```javascript
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 0);
}
// 3 3 3
```

`var` 是函数作用域，三个回调共享同一个 `i`，定时器执行时循环已经结束，`i` 变成了 3。

解决方式：

```javascript
for (let i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 0);
}
// 0 1 2
```

`let` 在每次循环中创建新的块级作用域，每个回调捕获不同的 `i`。

也可以用立即执行函数：

```javascript
for (var i = 0; i < 3; i++) {
    ((j) => {
        setTimeout(() => console.log(j), 0);
    })(i);
}
```

## 内存问题

闭包本身不是内存泄漏。问题在于闭包可能让本该释放的大对象一直可达。

```javascript
function bindHeavyData() {
    const bigData = new Array(100000).fill("*");

    return function logLength() {
        console.log(bigData.length);
    };
}

const fn = bindHeavyData();
```

只要 `fn` 还存在，`bigData` 就不会被回收。业务中常见风险包括：长期存在的事件监听、定时器、全局缓存引用 DOM 或大数据对象。

处理方式：

- 不需要时移除事件监听。
- 清理定时器。
- 避免闭包持有无用的大对象。
- 组件卸载时释放外部引用。

## 面试回答

可以这样答：

> 闭包是函数和它创建时词法环境的组合。一个内部函数即使在外部函数执行结束后，只要它还被引用，就仍然可以访问外部函数里的变量。原因是函数创建时会保留对外层词法作用域的引用，这些被引用的变量不会被垃圾回收。闭包常用于数据私有化、函数柯里化、保存异步上下文等。它的风险是可能让不再需要的变量长期留在内存里，所以事件监听、定时器、组件卸载时要注意清理引用。

## 高频追问

### 闭包一定会导致内存泄漏吗？

不会。闭包只是让函数保留外部变量的访问能力。只有当闭包长期持有不再需要的对象，并且没有释放引用时，才可能造成内存泄漏。

### 为什么 `let` 可以解决循环闭包问题？

因为 `let` 是块级作用域，循环每一轮都会创建新的词法环境，每个回调捕获的是当前轮次的变量。

### 闭包和作用域链是什么关系？

闭包依赖作用域链。内部函数通过作用域链访问外层变量，函数被带到外部执行时，这条词法作用域链仍然保留。
