# JavaScript 数据类型

## 面试定位

数据类型是 JavaScript 基础题里的入口题。面试官通常不是只想听“八种类型”，而是想继续追问：类型判断、存储差异、隐式转换、深浅拷贝、`null` 和 `undefined`、`Symbol` 和 `BigInt` 的使用场景。

## 核心原理

JavaScript 数据类型分为两类：

- 原始类型：`Undefined`、`Null`、`Boolean`、`Number`、`String`、`Symbol`、`BigInt`
- 引用类型：`Object`，包括普通对象、数组、函数、日期、正则等

原始类型保存的是值本身，赋值时通常复制值。引用类型保存的是对象引用，变量里存的是指向对象的引用地址，赋值时复制的是引用，所以多个变量可能指向同一个对象。

```javascript
let a = 1;
let b = a;
b = 2;
console.log(a); // 1

const obj1 = { count: 1 };
const obj2 = obj1;
obj2.count = 2;
console.log(obj1.count); // 2
```

这也是为什么原始类型比较看值，引用类型比较看引用地址。

```javascript
console.log(1 === 1); // true
console.log({} === {}); // false

const a = {};
const b = a;
console.log(a === b); // true
```

## 类型判断

### typeof

`typeof` 适合判断大部分原始类型和函数。

```javascript
typeof undefined; // "undefined"
typeof true; // "boolean"
typeof 1; // "number"
typeof "a"; // "string"
typeof Symbol(); // "symbol"
typeof 1n; // "bigint"
typeof function () {}; // "function"
```

但它有两个常见坑：

```javascript
typeof null; // "object"
typeof []; // "object"
```

`typeof null` 返回 `"object"` 是历史遗留问题。数组也是对象，所以不能用 `typeof` 区分数组和普通对象。

### instanceof

`instanceof` 判断对象是否出现在某个构造函数的原型链上。

```javascript
[] instanceof Array; // true
[] instanceof Object; // true
```

它的问题是只能判断引用类型，而且跨 iframe 或多执行上下文时可能失效，因为不同上下文里的 `Array` 构造函数不是同一个引用。

### Object.prototype.toString

更通用的判断方式：

```javascript
Object.prototype.toString.call([]); // "[object Array]"
Object.prototype.toString.call(null); // "[object Null]"
Object.prototype.toString.call(new Date()); // "[object Date]"
```

实际业务里，判断数组优先用 `Array.isArray()`。

## `null` 和 `undefined`

`undefined` 表示变量已声明但没有赋值，或者对象属性不存在、函数没有返回值。

`null` 表示主动赋值为空，通常用于表达“这里本来应该有对象，但现在没有”。

```javascript
let a;
console.log(a); // undefined

const user = null; // 明确表示暂无用户
```

面试回答时可以说：`undefined` 偏系统默认值，`null` 偏人为置空。

## Symbol 和 BigInt

`Symbol` 用来创建唯一值，常用于对象属性 key，避免命名冲突。

```javascript
const id = Symbol("id");
const user = {
    [id]: 1,
    name: "omen",
};
```

`BigInt` 用来表示超过 `Number.MAX_SAFE_INTEGER` 的整数。

```javascript
Number.MAX_SAFE_INTEGER; // 9007199254740991
const big = 9007199254740992n;
```

注意 `BigInt` 不能和普通 `Number` 直接混算。

```javascript
1n + 1; // TypeError
1n + BigInt(1); // 2n
```

## 面试回答

可以这样答：

> JavaScript 有 8 种数据类型，7 种原始类型和 1 种引用类型。原始类型包括 `undefined`、`null`、`boolean`、`number`、`string`、`symbol`、`bigint`，引用类型统一属于 `object`，数组、函数、日期这些本质上都是对象的不同表现。原始类型赋值时复制值，引用类型赋值时复制引用，所以对象赋值后修改属性会影响所有指向同一对象的变量。类型判断上，`typeof` 适合判断原始类型和函数，但 `typeof null` 是 `"object"`，数组也会返回 `"object"`；判断数组一般用 `Array.isArray()`，更完整的类型判断可以用 `Object.prototype.toString.call()`。

## 高频追问

### 为什么 `typeof null` 是 `"object"`？

这是 JavaScript 早期实现留下的历史问题，`null` 在底层标签上被误判成对象。现在为了兼容大量历史代码，这个行为保留了下来。

### `==` 和 `===` 有什么区别？

`===` 不做隐式类型转换，类型和值都相同才返回 `true`。`==` 会先做类型转换，规则复杂，容易产生反直觉结果。业务代码里建议优先使用 `===`。

```javascript
0 == false; // true
0 === false; // false
null == undefined; // true
null === undefined; // false
```

### 深拷贝和浅拷贝为什么和数据类型有关？

因为引用类型保存的是引用。浅拷贝只复制第一层属性，如果属性值仍然是对象，复制的仍是引用；深拷贝会递归复制嵌套对象。
