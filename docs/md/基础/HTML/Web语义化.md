# Web 语义化

## 面试定位

语义化是 HTML 基础高频题。面试重点是：为什么不用一堆 `div`，语义化对 SEO、无障碍、可维护性有什么价值，以及常见语义标签怎么用。

## 核心原理

Web 语义化是用合适的 HTML 标签表达内容结构和含义，而不是只用标签做样式容器。

例如页面主导航应该用 `nav`，独立文章用 `article`，主要内容用 `main`，页面头尾用 `header`、`footer`。

```html
<header>
    <h1>站点标题</h1>
    <nav>
        <a href="/">首页</a>
        <a href="/posts">文章</a>
    </nav>
</header>

<main>
    <article>
        <h2>文章标题</h2>
        <p>文章内容...</p>
    </article>
</main>

<footer>Copyright</footer>
```

## 常见语义标签

- `header`：页面或区块头部。
- `footer`：页面或区块底部。
- `main`：页面主体内容，一个页面通常只有一个。
- `nav`：导航区域。
- `article`：独立可分发内容，如文章、新闻、帖子。
- `section`：有主题的一组内容，通常配合标题。
- `aside`：补充内容，如侧边栏、相关推荐。
- `figure` / `figcaption`：图片、图表、代码块及说明。

## 价值

### 可维护性

语义标签让开发者更容易理解页面结构。

### SEO

搜索引擎可以更好理解页面内容层级和重要区域。

### 无障碍

屏幕阅读器可以根据语义结构帮助用户快速定位导航、主体和文章区域。

### 团队协作

结构清晰后，样式、测试、自动化抓取都更稳定。

## 面试回答

可以这样答：

> HTML 语义化就是用合适的标签表达内容含义，比如导航用 `nav`，主体用 `main`，独立文章用 `article`，页头页尾用 `header` 和 `footer`。它的价值不只是代码好看，还能提升可维护性，让搜索引擎更好理解页面结构，也能让屏幕阅读器这类辅助设备更准确地识别页面区域。语义化不是完全不用 `div`，而是在有明确含义的地方优先使用语义标签，普通布局容器仍然可以用 `div`。

## 高频追问

### section 和 article 区别？

`article` 表示独立完整内容，可以单独分发；`section` 表示页面中一个有主题的区域，通常需要标题。

### main 可以有多个吗？

一个页面通常只应有一个主要可见的 `main`，表示核心内容区域。

### 语义化是否会直接改变样式？

大多数语义标签默认表现和普通块级元素类似，语义化主要影响结构含义，不是为了样式。
