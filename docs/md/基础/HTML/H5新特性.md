# H5 新特性

## 面试定位

H5 新特性是 HTML 基础题。回答时不要只背 API 名称，要能按语义化、多媒体、表单、存储、通信、离线和图形能力分类说明。

## 核心原理

HTML5 不只是新增标签，它扩展了浏览器作为应用平台的能力，让 Web 能承载更复杂的交互和应用场景。

## 语义化标签

新增语义标签让页面结构更清晰：

- `header`
- `footer`
- `main`
- `nav`
- `article`
- `section`
- `aside`
- `figure`

详见 [Web 语义化](/md/基础/HTML/Web语义化.md)。

## 多媒体

HTML5 原生支持音视频，不再依赖 Flash。

```html
<video controls src="/video.mp4"></video>
<audio controls src="/audio.mp3"></audio>
```

常见能力：

- 播放控制。
- 字幕轨道。
- 多格式 source 兜底。

## 表单增强

新增输入类型：

- `email`
- `url`
- `number`
- `date`
- `range`
- `search`

新增属性：

- `placeholder`
- `required`
- `pattern`
- `autocomplete`
- `autofocus`

```html
<input type="email" required placeholder="请输入邮箱" />
```

## 存储能力

- `localStorage`：持久化本地存储。
- `sessionStorage`：会话级存储。
- `IndexedDB`：大量结构化数据存储。

注意：不要把敏感 token 明文放进 localStorage，容易被 XSS 读取。

## 通信能力

- `postMessage`：跨窗口、iframe 通信。
- `WebSocket`：实时双向通信。
- `Server-Sent Events`：服务端单向推送。

## 图形和能力 API

- `canvas`：像素级绘图，适合图表、游戏、图像处理。
- `svg`：矢量图形，适合图标、可缩放图形。
- Geolocation：地理位置。
- Drag and Drop：拖拽。
- Web Worker：后台线程。

## 面试回答

可以这样答：

> HTML5 的新特性可以分几类。第一是语义化标签，比如 `header`、`nav`、`main`、`article`，提升结构可读性、SEO 和无障碍。第二是多媒体，`audio`、`video` 原生支持音视频。第三是表单增强，新增 `email`、`number`、`date` 等类型和 `required`、`pattern` 等校验属性。第四是存储能力，比如 localStorage、sessionStorage、IndexedDB。第五是通信和应用能力，比如 postMessage、WebSocket、Web Worker、Canvas、地理定位等。实际项目里要注意兼容性和安全，比如敏感信息不要放 localStorage。

## 高频追问

### localStorage 和 sessionStorage 区别？

localStorage 持久保存，除非主动删除；sessionStorage 只在当前标签页会话内有效，标签页关闭后清除。

### HTML5 新表单校验能替代业务校验吗？

不能。前端校验只能提升体验，服务端仍必须校验，因为请求可以绕过浏览器直接发起。

### canvas 和 svg 怎么选？

canvas 适合大量图形、像素级绘制和频繁重绘；svg 适合矢量图、图标、少量可交互图形。
