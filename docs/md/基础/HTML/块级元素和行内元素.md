# 块级元素和行内元素

## 面试定位

块级和行内元素是 HTML/CSS 基础题。面试重点是默认布局行为、宽高是否生效、是否独占一行，以及 `inline-block`、`display` 改变格式化行为。

## 核心原理

HTML 元素的默认展示行为由 CSS 的 `display` 决定。常见类型包括：

- 块级元素：默认 `display: block`。
- 行内元素：默认 `display: inline`。
- 行内块元素：`display: inline-block`。

## 块级元素

特点：

- 独占一行。
- 默认宽度撑满父容器。
- 可以设置宽高、内外边距。
- 可以包含块级和行内内容，但具体要符合 HTML 内容模型。

常见元素：

- `div`
- `p`
- `h1` ~ `h6`
- `ul`
- `ol`
- `li`
- `section`
- `article`
- `header`
- `footer`

## 行内元素

特点：

- 不独占一行，和其他行内内容在同一行排列。
- 宽高通常不生效，尺寸由内容决定。
- 水平方向 margin/padding 生效；垂直方向对行高和布局影响有限。
- 一般只能包含文本或其他行内内容。

常见元素：

- `span`
- `a`
- `strong`
- `em`
- `label`
- `code`

## inline-block

`inline-block` 同时具备行内和块级特性：

- 不独占一行。
- 可以设置宽高。
- 可以设置完整盒模型。

```css
.tag {
    display: inline-block;
    width: 80px;
    height: 32px;
}
```

常用于标签、按钮、图标文本混排等场景。

## display 可以改变表现

元素语义和显示行为不是一回事。比如 `a` 是行内元素，但可以设置成块级表现。

```css
a.card-link {
    display: block;
}
```

## 面试回答

可以这样答：

> 块级元素默认独占一行，宽度通常撑满父容器，可以设置宽高和完整盒模型，比如 div、p、section。行内元素不会独占一行，宽高通常不生效，尺寸由内容决定，比如 span、a、strong。`inline-block` 不独占一行，但可以设置宽高，常用于按钮、标签这类需要在一行排列又要控制尺寸的元素。需要注意，元素语义和显示行为可以分开，默认行内的 a 也可以通过 `display: block` 改成块级布局。

## 高频追问

### img 是行内还是块级？

`img` 默认是 inline，但它是替换元素，可以设置宽高。

### 行内元素能设置 margin-top 吗？

可以写，但对普通行内布局的垂直空间影响有限。需要稳定控制尺寸时改成 `inline-block` 或 `block`。

### p 标签里能放 div 吗？

不应该。`p` 只能包含短语内容，放块级元素会导致浏览器自动闭合 `p`，结构可能和预期不同。
