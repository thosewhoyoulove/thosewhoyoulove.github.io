# Webpack 常见优化手段

Webpack 优化可以从构建速度和输出质量两个维度进行，下面分别介绍。

## 一、构建速度优化

### 1. 缩小搜索范围

通过合理配置 resolve 和 loader 的排除规则，减少文件搜索范围，提升构建速度。

```js
module.exports = {
  resolve: {
    // 指定第三方模块目录，减少搜索路径
    modules: [path.resolve(__dirname, 'node_modules')],
    // 明确扩展名，减少尝试次数
    extensions: ['.js', '.jsx', '.json'],
    // 配置别名
    alias: {
      '@': path.resolve(__dirname, 'src')
    },
    // 指定 package.json 中的入口字段
    mainFields: ['main']
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        // 排除 node_modules
        exclude: /node_modules/,
        use: 'babel-loader'
      }
    ]
  }
};
```

### 2. 使用缓存

#### 2.1 babel-loader 缓存

```js
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            cacheDirectory: true // 开启缓存
          }
        }
      }
    ]
  }
};
```

#### 2.2 持久化缓存（Webpack 5+）

```js
module.exports = {
  cache: {
    type: 'filesystem', // 使用文件系统缓存
    buildDependencies: {
      config: [__filename] // 配置文件变化时失效
    }
  }
};
```

### 3. 多进程构建

#### 3.1 thread-loader

将耗时的 loader 放到多进程池中运行。

```js
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: [
          {
            loader: 'thread-loader',
            options: {
              workers: require('os').cpus().length - 1 // CPU 核心数 - 1
            }
          },
          'babel-loader'
        ]
      }
    ]
  }
};
```

#### 3.2 parallel-webpack / HappyPack（已废弃）

> 注意：HappyPack 已不再维护，推荐使用 thread-loader。

### 4. 开发环境优化

#### 4.1 开启 DllPlugin（适用于 Webpack 4）

将第三方库预先打包成动态链接库，减少每次构建的时间。

```js
// webpack.dll.config.js
const path = require('path');
const webpack = require('webpack');

module.exports = {
  entry: {
    vendor: ['react', 'react-dom', 'lodash']
  },
  output: {
    path: path.resolve(__dirname, 'dll'),
    filename: '[name].dll.js',
    library: '[name]_library'
  },
  plugins: [
    new webpack.DllPlugin({
      path: path.resolve(__dirname, 'dll/[name]-manifest.json'),
      name: '[name]_library'
    })
  ]
};

// webpack.config.js
const webpack = require('webpack');

module.exports = {
  plugins: [
    new webpack.DllReferencePlugin({
      manifest: require('./dll/vendor-manifest.json')
    })
  ]
};
```

> 注意：Webpack 5 的持久化缓存已替代 DllPlugin，通常不再需要。

#### 4.2 合理使用 source-map

不同环境选择合适的 source-map 类型：

| 环境 | 推荐 source-map | 说明 |
| --- | --- | --- |
| 开发环境 | eval-cheap-module-source-map | 构建快，映射到源码行 |
| 生产环境 | (none) 或 hidden-source-map | 不暴露源码，仅用于错误上报 |

```js
module.exports = {
  devtool: process.env.NODE_ENV === 'development' 
    ? 'eval-cheap-module-source-map' 
    : 'hidden-source-map'
};
```

## 二、输出质量优化

### 1. 代码压缩

#### 1.1 JavaScript 压缩

Webpack 5 内置了 TerserWebpackPlugin，默认开启。

```js
const TerserPlugin = require('terser-webpack-plugin');

module.exports = {
  optimization: {
    minimize: true,
    minimizer: [
      new TerserPlugin({
        parallel: true, // 开启多进程
        terserOptions: {
          compress: {
            drop_console: true // 移除 console
          }
        }
      })
    ]
  }
};
```

#### 1.2 CSS 压缩

```js
const CssMinimizerPlugin = require('css-minimizer-webpack-plugin');

module.exports = {
  optimization: {
    minimizer: [
      new CssMinimizerPlugin()
    ]
  }
};
```

### 2. 代码分割（Code Splitting）

#### 2.1 SplitChunksPlugin

自动提取公共模块和第三方库。

```js
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all', // 对所有模块进行分割
      minSize: 20000, // 最小分割大小
      minChunks: 1, // 最少被引用次数
      cacheGroups: {
        vendors: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all'
        },
        commons: {
          name: 'commons',
          minChunks: 2,
          chunks: 'initial'
        }
      }
    }
  }
};
```

#### 2.2 动态导入

使用 `import()` 语法实现按需加载。

```js
// 路由懒加载
const Home = () => import('./pages/Home');
const About = () => import('./pages/About');

// 或者使用魔法注释指定 chunk 名称
const Dashboard = () => import(/* webpackChunkName: "dashboard" */ './pages/Dashboard');
```

### 3. Tree Shaking

移除未使用的代码（需要 ES Module）。

```js
module.exports = {
  mode: 'production', // 生产模式自动开启
  optimization: {
    usedExports: true, // 标记未使用的导出
    sideEffects: true // 配合 package.json 的 sideEffects 字段
  }
};
```

在 `package.json` 中配置：

```json
{
  "sideEffects": false
}
```

对于有副作用的文件（如全局 CSS）：

```json
{
  "sideEffects": ["*.css", "*.scss"]
}
```

### 4. Scope Hoisting

将模块合并到一个函数作用域中，减少函数声明和内存开销。

```js
module.exports = {
  mode: 'production', // 生产模式自动开启
  optimization: {
    concatenateModules: true
  }
};
```

### 5. 资源优化

#### 5.1 图片压缩

```js
const ImageMinimizerPlugin = require('image-minimizer-webpack-plugin');

module.exports = {
  optimization: {
    minimizer: [
      new ImageMinimizerPlugin({
        minimizer: {
          implementation: ImageMinimizerPlugin.imageminMinify,
          options: {
            plugins: [
              ['gifsicle', { interlaced: true }],
              ['jpegtran', { progressive: true }],
              ['optipng', { optimizationLevel: 5 }],
              ['svgo', { plugins: [{ name: 'removeViewBox', active: false }] }]
            ]
          }
        }
      })
    ]
  }
};
```

#### 5.2 小图片转 Base64

```js
module.exports = {
  module: {
    rules: [
      {
        test: /\.(png|jpg|gif)$/,
        type: 'asset',
        parser: {
          dataUrlCondition: {
            maxSize: 8 * 1024 // 小于 8KB 转 Base64
          }
        }
      }
    ]
  }
};
```

### 6. 提取 CSS 到单独文件

```js
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        use: [
          MiniCssExtractPlugin.loader, // 替代 style-loader
          'css-loader'
        ]
      }
    ]
  },
  plugins: [
    new MiniCssExtractPlugin({
      filename: 'css/[name].[contenthash:8].css'
    })
  ]
};
```

## 三、加载性能优化

### 1. 文件名哈希

利用浏览器缓存，只有内容变化时文件名才变化。

```js
module.exports = {
  output: {
    filename: '[name].[contenthash:8].js',
    chunkFilename: '[name].[contenthash:8].chunk.js'
  }
};
```

### 2. 外部扩展（Externals）

将大型库从 bundle 中排除，通过 CDN 加载。

```js
module.exports = {
  externals: {
    react: 'React',
    'react-dom': 'ReactDOM',
    lodash: '_'
  }
};
```

HTML 中引入 CDN：

```html
<script src="https://cdn.jsdelivr.net/npm/react@18/umd/react.production.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/react-dom@18/umd/react-dom.production.min.js"></script>
```

### 3. 预加载和预获取

```js
// 预加载：当前页面可能需要的资源（优先级高）
import(/* webpackPrefetch: true */ './module');

// 预获取：未来可能需要的资源（优先级低，空闲时加载）
import(/* webpackPreload: true */ './module');
```

### 4. Gzip 压缩

```js
const CompressionPlugin = require('compression-webpack-plugin');

module.exports = {
  plugins: [
    new CompressionPlugin({
      algorithm: 'gzip',
      test: /\.(js|css|html|svg)$/,
      threshold: 10240, // 大于 10KB 才压缩
      minRatio: 0.8
    })
  ]
};
```

## 四、分析工具

### 1. Webpack Bundle Analyzer

可视化分析打包后的文件大小。

```js
const BundleAnalyzerPlugin = require('webpack-bundle-analyzer').BundleAnalyzerPlugin;

module.exports = {
  plugins: [
    new BundleAnalyzerPlugin({
      analyzerMode: 'static'
    })
  ]
};
```

### 2. Speed Measure Plugin

测量各 loader 和 plugin 的耗时。

```js
const SpeedMeasurePlugin = require('speed-measure-webpack-plugin');
const smp = new SpeedMeasurePlugin();

module.exports = smp.wrap({
  // webpack 配置
});
```

### 3. Webpack Profile

通过命令行生成构建报告。

```bash
webpack --profile --json > stats.json
```

然后上传到 [Webpack Analyse](https://webpack.github.io/analyse/) 或 [Statoscope](https://statoscope.tech/) 进行分析。

## 五、优化总结

| 优化类型 | 优化手段 | 效果 |
| --- | --- | --- |
| 构建速度 | 缩小搜索范围 | 减少文件查找时间 |
| 构建速度 | 开启缓存 | 避免重复编译 |
| 构建速度 | 多进程构建 | 利用多核 CPU |
| 构建速度 | 合理的 source-map | 减少映射生成时间 |
| 输出质量 | 代码压缩 | 减小文件体积 |
| 输出质量 | 代码分割 | 减小首屏加载体积 |
| 输出质量 | Tree Shaking | 删除未使用代码 |
| 输出质量 | Scope Hoisting | 减少函数声明 |
| 输出质量 | 资源优化 | 图片压缩、Base64 |
| 加载性能 | 文件名哈希 | 利用浏览器缓存 |
| 加载性能 | Externals | 减小 bundle 体积 |
| 加载性能 | 预加载/预获取 | 优化加载时机 |
| 加载性能 | Gzip 压缩 | 减小传输体积 |
