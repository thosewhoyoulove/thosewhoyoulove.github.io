# HMR 热更新原理

## 面试定位

HMR 题通常和 Webpack、Vite 对比一起考。回答重点是：文件变化如何通知浏览器、模块如何替换、什么时候会退化成整页刷新。

## 面试回答

> HMR 的目标是在不刷新整个页面的情况下替换变更模块。开发服务器监听文件变化后，会更新模块图，找到能接受热更新的边界，然后通过 WebSocket 通知浏览器。浏览器收到消息后拉取新的模块代码，并执行框架或模块声明的 accept 回调。Webpack 的 HMR 基于 bundle runtime、module id 和热更新 chunk；Vite 基于原生 ESM，浏览器重新请求带时间戳的变更模块，所以链路更轻。如果某个模块没有 HMR 边界，更新会继续向上冒泡，最后可能退化成整页刷新。

一句话总结：

> HMR = 文件监听 + 模块图定位 + WebSocket 通知 + 浏览器拉取新模块 + 热更新边界接管。

## 核心原理

```text
文件变化
  -> dev server 监听到变更
  -> 更新模块图
  -> 找到 HMR 边界
  -> WebSocket 通知浏览器
  -> 浏览器请求新模块
  -> 执行 accept 回调或框架运行时更新
```

Webpack 和 Vite 的差异：

| 工具 | HMR 方式 | 特点 |
| --- | --- | --- |
| Webpack | runtime 加载热更新 chunk | 和 bundle 体系绑定 |
| Vite | 浏览器重新 import ESM 模块 | 不需要重新打包整个应用 |

模块边界示意：

```text
App.vue / App.jsx
  -> Button
  -> Chart

修改 Button：
如果框架插件能处理组件热更新，只替换 Button；
如果不能处理，向上冒泡到 App；
如果仍没有 accept，整页刷新。
```

## 高频追问

### HMR 为什么能保持状态？

不是所有 HMR 都能保持状态。React/Vue 组件通常由框架插件判断是否能安全替换组件逻辑，并保留运行时状态。普通模块没有边界时可能无法保留。

### Vite HMR 为什么快？

Vite 开发态不打包整个项目。修改文件后只需要让浏览器重新请求变更模块和相关边界，不需要重新生成完整 bundle。

### HMR 和 Live Reload 区别是什么？

Live Reload 是文件变化后刷新整个页面。HMR 是替换局部模块，尽量保留应用状态。
