# 你的项目经历（重点）

这类问题考察：**你是否真实参与、能否讲清架构与取舍、能否扛住追问**。回答时用「**背景 → 职责 → 方案 → 难点 → 数据结果**」结构，控制在 2～3 分钟，预留追问接口。

---

## 提问示例（结合本站项目笔记）

- 你的 [WebRTC 会议室项目](/md/面试准备/项目与架构/WebRTC%20会议室项目.md) 是如何实现音视频通信的？若扩展到千人会议，架构上要如何优化？
- 你的 [智慧大屏数据可视化](/md/面试准备/项目与架构/智慧大屏数据可视化.md) 如何做到高性能渲染？数据量很大时如何防止浏览器崩溃？
- 你的 [仿网易云音乐移动端](/md/面试准备/项目与架构/仿网易云音乐移动端.md)（若尚未成文，可先口述架构）如何做播放状态管理？跨页面保持播放？
- 你的 [面对面签到系统](/md/面试准备/项目与架构/面对面签到系统.md)（同上）如何防伪造签到？用过哪些策略？

---

## 应答建议

1. **选一个最熟的项目**展开，不要每个都浅尝辄止。  
2. **数字**：延迟、QPS、包体积、帧率、错误率，有一两个即可。  
3. **诚实**：未做的部分如实说，并补「若再做会如何设计」。  
4. **对齐文档**：上述带链接的专题尽量与口头叙述一致，避免简历、博客、口述三套说法。

若某项目尚未写成文章，可在 `面试准备/项目与架构/` 下补充一篇，便于日后迭代。
