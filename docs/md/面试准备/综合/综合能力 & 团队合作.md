# 综合能力 & 团队合作

这类问题没有标准答案，面试官想看你的**沟通方式、冲突处理、质量意识、取舍能力**。可用 **STAR**（情境–任务–行动–结果）组织 1～2 分钟回答。

---

## 你在团队开发中有没有遇到技术分歧？你是如何解决？

**思路**：不贬低对方方案 → 对齐目标（性能/工期/风险）→ 列出选项与权衡 → 数据或 PoC 验证 → 达成共识并记录决策。  
**例**：CSR 还是 SSR、状态放 Redux 还是局部组件、是否引入新依赖 —— 用首屏指标、包体积、维护成本说事。

---

## 你有过 Code Review 经验吗？你如何保证团队代码质量？

**可做**：约定 MR 模板、关键点（安全、边界、测试、可维护性）、小步合并、静态检查与 CI 卡点、定期复盘典型问题。  
**态度**：Review 是对事不对人，提建议并附参考链接或示例 patch。

---

## 需求有多种实现方式时，如何做决策？

**框架**：约束（时间、人力、兼容性）→ 各方案**成本/风险/可逆性** → 对核心指标的影响 → 是否需 AB 或灰度。  
可说：优先可演进方案，避免「一次性黑科技」拖垮后期。

---

## 遇到过最有挑战性的问题？如何解决的？

**结构**：问题现象 → 根因分析过程（日志、复现、工具）→ 方案与权衡 → 结果（数据）→ 复盘（监控、文档、防再发）。  
技术栈尽量与简历项目一致，避免空泛。

---

## 与跨角色（产品、测试、后端）协作的注意点

**产品**：尽早澄清验收标准与边界，变更走评估。  
**测试**：可测性（日志、开关、测试环境）、缺陷分级与回归范围。  
**后端**：接口契约、错误码、超时与幂等、联调窗口。
