# 团队协作与主导能力

偏「带队 / 推动落地」的叙述题，建议用**真实项目**说明职责边界与可量化结果。

---

## 1. 你是怎么带领团队开发核心产品的？

我负责项目规划、模块拆分、进度管理和关键功能实现。在需求阶段与产品一起梳理原型，开发阶段使用 Git Flow 和 Jira 协作，定期 Code Review 与 Daily Standup。遇到 UI 实现难题或接口不明确时，主动协调前后端沟通，确保按期交付。

**补充可讲**：风险清单（技术债、依赖方延期）、里程碑汇报、在阻塞点上的升级路径（何时找 leader）。

---

## 2. 项目如何保证质量？是否做测试？

我们使用 Vitest 编写单元测试，接口用 Postman / 自动化脚本回归。上线前走灰度发布，保障稳定性。测试覆盖率目标约 70%，核心组件与关键路径优先覆盖。

**可延伸**：E2E 的取舍、契约测试、线上可观测（错误监控、性能指标）、发布 checklist。

---

## 3. 如何分配任务与跟踪进度？

按模块与人群技能匹配任务，**WBS** 拆到可交付粒度；每日/双日同步进度与阻塞；用看板暴露风险；对长耗时任务设检查点，避免临期爆雷。

---

## 4. 新人 onboarding 或知识传承你怎么做？

文档化（架构图、环境搭建、常见问题）、结对前几次 MR、`good first issue` 式小任务、定期内部分享；重要决策写入 ADR 或 wiki，减少口口相传。

---

## 5. 与更高层预期不一致时怎么办？

用事实依据（工时评估、风险、业务优先级）沟通；给出**多套方案**（快/稳/省）让 decision maker 选；确认后调整计划并同步全员，避免隐含期望。
