# 全栈岗位冲刺路线（千帆 / DuMate）

## 面试定位

岗位从「前端研发」调整为 **DuMate-全栈研发工程师**（千帆公有云前端组）。面试官会同时看：**React 深度、Node/Go 协作边界、Agent/Harness 工程化意识、端到端落地与产品 sense**。

本路线面向 **面试前 1～2 天**，在现有前端复习基础上做增量，不重复刷已熟练的八股。

## 核心原理

全栈岗不是「前端 + 会写接口」，而是能在 **用户视角 → 产品方案 → 前后端协作 → 稳定性与体验** 一条链路上负责关键段。千帆/DuMate 场景还会追问 **智能体、Harness、Token 成本、沙箱安全**。

```text
岗位匹配
  → 业务认知（DuMate / 千帆 / Agent）
  → 技术栈（React 深 + Node 协作 + Go 概念）
  → 工程化（Harness、MCP、评测、迭代）
  → 项目闭环（问题-方案-结果-复盘）
  → 百度面经对齐
```

## 第 3 步：项目与产品 sense（约 2 小时）

每个核心项目准备 **2 分钟 + 5 分钟** 版本，并加一句 **用户体验/效率提升**：

- [你的项目经历（重点）](/md/面试准备/项目与架构/你的项目经历（重点）.md)
- [高级筛选系统](/md/面试准备/项目与架构/高级筛选系统.md)（体现复杂 B 端与架构）
- 任选 1 个：WebRTC / 大屏 / 架构升级

**产品 sense 话术**：不只「做完功能」，要说 **用户是谁、痛点、为什么这样设计、如何验证效果**。

---

## 第 4 步：模拟追问（约 1 小时）

用 [高频追问清单](/md/面试准备/高频追问清单.md) 自测，额外自问：

- DuMate 是什么？和千帆什么关系？
- Harness 工程解决什么问题？你在开发里怎么体现？
- React + Node + Go 在你项目里怎么分工？
- 如何让 Agent 任务更稳定、更省 Token？
- 前端在智能体产品里负责什么？

---

## 面试回答

若被问「岗位变了你怎么准备」：

> 我按全栈研发重新梳理了准备重点：React 和工程化是我强项，会结合百度面经把 Hooks、渲染和状态管理答深；协作上补强了 Node BFF、API 设计和鉴权边界，并准备了 Go 在高性能服务里的定位说明；业务上了解了 DuMate 作为千帆体系里的通用智能体，以及 Harness 工程通过规则、工具链和反馈闭环提升 Agent 成功率、控制 Token 的思路。项目表达上我会强调端到端落地和用户体验指标，而不只讲页面实现。
