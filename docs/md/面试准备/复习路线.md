# 复习路线

## 面试定位

这份路线不是按知识点完整性排序，而是按面试命中率和提分效率排序。

复习时不要只追求“看完”，而要保证每个高频问题都能答出四层内容：

1. 结论。
2. 原理。
3. 项目场景。
4. 边界取舍。

## 核心原理

前端面试通常会从基础问题进入，再逐步深挖到框架、工程化、性能和项目。

所以复习顺序应该是：

```text
基础概念 -> 浏览器与网络 -> 框架原理 -> 工程化与性能 -> 项目表达 -> 模拟追问
```

如果时间有限，优先保证能回答高频题和项目深挖题。低频细节可以后补。

### 第 1 天

目标是补基础和定位薄弱点。

- JavaScript 数据类型、this、闭包、Promise、事件循环。
- TypeScript 基础类型、泛型、类型守卫。
- 用追问清单标出不会答的问题。

### 第 2 天

目标是补框架和浏览器。

- Vue 响应式、Diff、Vue 2 与 Vue 3 区别。
- React Hooks、状态更新、组件性能优化。
- 浏览器渲染、缓存、首屏优化。

### 第 3 天

目标是补项目表达和模拟面试。

- 每个项目准备 2 分钟版本。
- 每个核心项目准备 5 分钟版本。
- 每个技术点都补一句“我在项目里怎么用过”。

## 面试回答

如果面试官问我怎么准备前端面试，我会按命中率来安排。

第一阶段先补 JavaScript、TypeScript、浏览器、网络这些基础，因为这些是所有公司都会问的。第二阶段准备 Vue 或 React 的核心原理，比如响应式、Diff、Hooks、状态更新。第三阶段补工程化和性能优化，因为这部分能体现项目经验和体系化能力。最后一定要花时间准备项目表达，把每个项目讲成“问题、方案、结果、复盘”的闭环。

我不会只背答案，而是会用追问清单模拟面试。每个问题至少答出结论、原理、项目场景和边界取舍，这样面试官继续追问时才接得住。

## 高频追问

### 时间很少时先复习什么？

先复习 JavaScript、浏览器、网络、框架核心原理和项目表达。

这几块命中率最高，也最容易被连续追问。

### 怎么判断自己是否复习到位？

不要用“看过”判断，要用“能不能讲出来”判断。

每个问题能在 1 分钟内讲出结论和原理，能再补一个项目场景，基本就达到可面试状态。

### 项目和八股哪个更重要？

两者都重要，但作用不同。

基础题决定你能不能过技术筛选，项目题决定你能不能体现经验深度和岗位匹配度。
