# 前端性能优化

## 面试定位

性能优化题考察的不是你能背多少手段，而是你能不能从指标出发定位瓶颈，再选择合适方案。

面试里最好的回答不是“我做了懒加载、压缩、CDN”，而是：

```text
指标是什么 -> 瓶颈在哪里 -> 做了什么优化 -> 数据有什么变化 -> 有什么代价
```

## 核心原理

前端性能可以按链路拆成四段：

1. 网络加载：HTML、JS、CSS、图片、字体、接口。
2. 解析执行：JS 解析、编译、执行，CSSOM 和 DOM 构建。
3. 渲染展示：布局、绘制、合成、首屏内容出现。
4. 交互响应：主线程是否阻塞，输入是否及时响应。

常见指标包括：

- FCP：首次内容绘制。
- LCP：最大内容绘制，衡量首屏核心内容出现速度。
- CLS：布局偏移，衡量视觉稳定性。
- INP：交互响应，衡量用户操作后的响应延迟。
- TTFB：首字节时间，偏服务端和网络。

## 首屏优化

首屏慢通常从这些方向排查：

- HTML 是否需要等待服务端太久。
- JS 包体积是否过大。
- CSS 是否阻塞渲染。
- 图片和字体是否过大。
- 接口是否串行阻塞首屏。
- 第三方脚本是否占用主线程。

常见方案：

- 路由和组件懒加载。
- 关键资源 `preload`。
- 非关键资源延迟加载。
- 图片压缩、WebP/AVIF、响应式图片。
- CDN 和缓存。
- 代码分包和 Tree Shaking。
- SSR/SSG 提前输出首屏 HTML。

## 代码分包

代码分包的目标是减少首屏必须加载的 JS。

常见方式：

- 路由级动态 import。
- 组件级懒加载。
- vendor 包拆分。
- 大型库按需加载。

但分包不是越细越好。分包太细会增加请求数量和调度成本，也可能造成运行时瀑布请求。

面试可以这样答：

```text
我会先用构建分析工具看包体积，再把首屏不需要的页面、组件和大型依赖拆出去。
拆完后还要看网络瀑布图，避免分包过细导致请求链路变长。
```

## 渲染和交互优化

如果页面加载完但操作卡顿，通常是主线程被长任务占用。

常见方案：

- 减少不必要的渲染。
- 使用虚拟列表处理大数据列表。
- 把重计算缓存或移到 Web Worker。
- 合理使用防抖和节流。
- 避免频繁读写布局导致强制同步布局。
- 动画优先使用 transform 和 opacity。

React 项目关注无效重渲染，Vue 项目关注响应式数据粒度和列表 key。

## 缓存优化

静态资源通常使用强缓存：

```text
文件名带 hash -> Cache-Control 设置较长 max-age
```

HTML 通常不做长期强缓存，避免发版后入口文件不更新。

接口缓存要看业务实时性。实时性低的数据可以缓存，权限、订单、交易类数据要谨慎。

## 项目案例回答

如果讲项目优化，可以这样组织：

```text
当时页面的主要问题是首屏加载慢，LCP 大约是【数值】。
我先通过 Performance 和构建分析定位，发现主要瓶颈是【包体积/图片/接口/长任务】。
然后做了【代码分包、图片压缩、缓存、接口并行、组件懒加载】。
优化后 LCP 从【A】降到【B】，首屏 JS 从【A】降到【B】。
```

没有真实数据时，也要说验证方式，比如 Lighthouse、Performance 面板、Network 瀑布图、线上监控。

## 面试回答

我做性能优化时会先看指标，不会直接堆优化手段。

如果是首屏慢，我会先看 TTFB、资源加载瀑布图、JS 包体积、图片体积和接口是否阻塞首屏。然后针对瓶颈选择方案，比如代码分包、图片压缩、CDN 缓存、关键资源预加载、接口并行、SSR 或 SSG。

如果是交互卡顿，我会看主线程有没有长任务，组件是否频繁渲染，列表是否过大，是否有大量同步计算。对应方案可能是虚拟列表、减少无效渲染、Web Worker、节流防抖、缓存计算结果。

我认为性能优化一定要有闭环：优化前采样，优化中定位，优化后复测，上线后继续看真实用户数据。

## 高频追问

### 首屏加载慢怎么排查？

先看 Network 和 Performance。

Network 看 TTFB、资源大小、请求是否串行、缓存是否命中。Performance 看主线程任务、渲染时机、LCP 元素和长任务。

### 代码分包有什么副作用？

分包会减少首屏包体积，但可能增加请求数量。

如果拆得太细，可能导致瀑布请求，反而变慢。所以要结合 HTTP/2、预加载和实际访问路径设计分包。

### 图片怎么优化？

压缩体积，使用 WebP/AVIF，按设备尺寸加载响应式图片，非首屏图片懒加载，重要图片可以 preload。

还要设置宽高，避免图片加载后造成布局偏移。

### 怎么证明性能优化有效？

使用同一口径对比优化前后数据。

本地可以用 Lighthouse、Performance、Network；线上可以看真实用户监控，包括 LCP、CLS、INP、错误率和资源加载耗时。
