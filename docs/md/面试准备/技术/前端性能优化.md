## 你在项目中做过哪些前端性能优化？可以举例说明吗？

**推荐结构**：选 **1～2 个指标**（LCP、FID/INP、TTI、CLS）→ 对应手段 → 前后数据。

## WebRTC 的首帧渲染优化你是怎么做的？（针对你的 WebRTC 项目）

**可答方向**：信令尽早建立、`getUserMedia` 并行、编解码器协商缓存、降低初始分辨率再自适应、TURN 选型减少建连失败、首帧前占位 UI、统计 `RTT`/丢包驱动码率。结合你文档 [WebRTC 会议室项目](/md/面试准备/项目与架构/WebRTC%20会议室项目.md) 展开真实数据。

## 智慧大屏构建体积从 12MB 降到 3.5MB，你是怎么优化的？

**常见组合**：按需引入图表库与组件库、外部化大型依赖或放 CDN、`rollup`/`vite` 构建分析（`rollup-plugin-visualizer`）、删除未用 polyfill、关闭 source map 上线、压缩与拆包、若用 ECharts 可仅注册所用图表类型。详见 [智慧大屏数据可视化](/md/面试准备/项目与架构/智慧大屏数据可视化.md)。

## 如何减少页面的首次加载时间（TTFB）？

**TTFB** 偏服务端与网络：CDN 边缘、HTTP/2/3、缓存动态页（短时缓存/Edge）、数据库与 API 优化、压缩、TLS 会话复用、地域就近部署。  
前端可配合：**尽早可缓存的 HTML**、减少重定向、预连接 `preconnect`、HTTP 缓存策略。

## 你在 Next.js 或 Vue 里用过 Code Splitting 吗？是如何实现的？

**思路**：按路由动态 `import()`、组件懒加载（Vue `defineAsyncComponent`、React `lazy` + `Suspense`）、Webpack Magic Comments、`splitChunks` 提取 vendor。说明**权衡**：请求数变多需配合 HTTP/2 与预加载。

## PWA 的核心技术有哪些？如何让 Web App 实现离线访问？

**核心**：`Web App Manifest`、`Service Worker`（install/fetch/cache）、**HTTPS**、缓存策略（Cache First/Network First 等）、`Workbox`（可选）。  
**离线**：SW 拦截请求，命中 Cache Storage；注意版本更新与缓存清理。
