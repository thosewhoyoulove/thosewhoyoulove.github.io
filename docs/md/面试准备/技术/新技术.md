# 前端新技术

## 面试定位

新技术问题不是让你罗列热点名词，而是考察你是否有技术判断力。

面试官通常想看：

1. 你是否关注技术趋势。
2. 你是否理解新技术解决什么问题。
3. 你是否能判断适不适合项目。
4. 你是否知道引入新技术的成本和风险。

回答时不要说“这个很火，所以要用”。要说清楚收益、代价和落地条件。

## 核心原理

评估新技术可以用四个维度：

- 解决什么问题：性能、体验、效率、架构复杂度、协作成本。
- 成熟度如何：生态、社区、文档、维护频率、生产案例。
- 迁移成本多高：学习成本、改造范围、兼容性、测试成本。
- 风险是否可控：回滚方案、灰度策略、团队掌握程度。

技术选型的本质是权衡，不是追新。

## 框架演进

### Vue 3

Vue 3 的重点不是 Composition API 这个语法本身，而是更好的逻辑复用、TypeScript 支持和响应式能力。

适合场景：

- 复杂组件逻辑需要按功能组织。
- 项目需要更好的 TS 推导。
- 希望利用 Vue 3 生态，如 Pinia、Vite、Teleport。

迁移时要关注 Vue 2 插件兼容、生命周期变化、`v-model` 变化、全局 API 变化。

### React 18

React 18 的重点是并发能力、自动批处理、`startTransition`、Suspense 生态演进。

它解决的是复杂交互场景下渲染优先级和用户体验问题。

面试里可以说：并发渲染不是让代码多线程运行，而是让 React 可以根据优先级调度渲染工作。

### Svelte

Svelte 是编译时框架，很多工作在编译阶段完成，运行时负担较小。

它的价值是更少运行时代码和更接近原生的输出。但生态、团队熟悉度和大型项目实践仍然是选型时要考虑的成本。

## 工程和架构趋势

### Vite

Vite 的核心价值是改善开发体验。开发态利用原生 ESM 和依赖预构建，启动和热更新更快。

生产环境仍然需要考虑 Rollup 打包、分包、缓存和兼容性。

### 微前端

微前端适合大型系统、多团队并行开发、技术栈不完全统一的场景。

它解决的是组织和部署复杂度，不是性能优化银弹。

代价包括：

- 子应用通信复杂。
- 公共依赖治理困难。
- 样式和运行时隔离成本。
- 首屏资源可能变多。
- 运维和监控更复杂。

### Serverless

Serverless 适合事件驱动、低频接口、轻量 BFF、定时任务、图片处理等场景。

优点是不用管理服务器、按量付费、扩缩容方便。

代价是冷启动、平台绑定、调试复杂、长连接和复杂状态处理不一定适合。

## Web 能力趋势

### WebAssembly

WebAssembly 适合高计算场景，比如图像处理、音视频处理、游戏、CAD、加解密、科学计算。

它不是替代 JavaScript，而是让 Web 能运行更接近原生性能的模块。

### Web Components

Web Components 是浏览器原生组件标准，适合跨框架组件共享。

但在复杂状态管理、工程生态、样式协作方面，仍然要看项目是否真的需要跨框架复用。

### PWA

PWA 通过 Service Worker、Manifest、缓存策略实现离线能力和类 App 体验。

适合内容型、低网环境、需要离线访问的应用。要注意缓存更新、版本管理和兼容性。

## AI 与前端

AI 和前端结合主要有两类：

- 面向用户：智能客服、搜索推荐、文本生成、图像识别、语音交互。
- 面向研发：代码生成、测试生成、文档总结、UI 还原、辅助排查。

落地时要注意数据安全、接口成本、响应延迟、模型幻觉和失败兜底。

## 面试回答

我关注新技术时不会只看热度，而是先看它解决什么问题。

比如 Vite 解决的是开发环境启动慢和热更新慢的问题；React 18 的并发能力解决的是复杂交互下渲染优先级的问题；Vue 3 的 Composition API 更适合复杂逻辑复用和 TypeScript；微前端解决的是大型组织下多团队独立开发和部署的问题。

但每个新技术都有成本。引入前我会评估成熟度、团队学习成本、迁移范围、兼容性、监控和回滚方案。如果只是为了追新，而没有明确收益，就不应该贸然引入。比较稳妥的方式是先在低风险模块 PoC，再根据指标和维护成本决定是否推广。

## 高频追问

### 如何判断一个新技术能不能用于生产？

看成熟度、生态、文档、社区活跃度、生产案例、团队掌握程度和风险兜底。

还要做 PoC，用真实业务场景验证，而不是只看 Demo。

### 微前端什么时候值得用？

当系统很大、团队很多、发布节奏不同、技术栈不统一时值得考虑。

如果只是一个中小项目，微前端可能会引入不必要的复杂度。

### WebAssembly 会替代 JavaScript 吗？

不会。

Wasm 更适合高性能计算模块，JavaScript 仍然负责业务逻辑、DOM 操作和生态集成。

### 面试问你最近关注什么新技术怎么答？

选一个你能讲清楚原理和场景的技术，不要贪多。

回答结构是：它解决什么问题、核心机制是什么、适合什么场景、有什么代价、你会怎么落地验证。
