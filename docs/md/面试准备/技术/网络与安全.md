## 你在前端项目中遇到过哪些安全性问题？你是如何解决的？

**答题思路**：按场景说清「风险 → 发现方式 → 措施 → 验证」。  
可举：接口未鉴权导致越权、富文本 XSS、第三方脚本劫持、`postMessage` 未校验来源、`localStorage` 存敏感信息、依赖供应链漏洞等。每项对应：输入输出编码、CSP、Token 与 SameSite、HTTPS、权限最小化、依赖审计（`npm audit`）等。

## 你能解释一下 CSRF 和 XSS 吗？如何防止 XSS 攻击？

**XSS**：恶意脚本注入页面，在用户浏览器执行，可窃取 Cookie、改 DOM、伪造操作。  
防御：输出编码、CSP、`HttpOnly` Cookie、避免 `innerHTML` 拼不可信数据、富文本白名单过滤。

**CSRF**：诱导已登录用户在**目标站点**发起非本意请求（利用浏览器自动带 Cookie）。  
防御：CSRF Token、SameSite Cookie、关键操作二次校验、校验 `Origin`/`Referer`。

## 在 WebRTC 项目中，如何防止 IP 泄露和数据劫持？

**思路**：  
- 信令与媒体通道走 **HTTPS/WSS**，防窃听与中间人。  
- **TURN** 中继可隐藏真实 IP（视架构与需求，STUN 仍可能暴露，需结合产品说明）。  
- **DTLS-SRTP** 用于媒体加密；信令侧鉴权、房间/令牌时效、防重放。  
- 不将敏感元数据写入可被 XSS 读到的存储。

## HTTP 缓存策略有哪些？`Cache-Control: max-age=0, must-revalidate` 代表什么？

**常见**：强缓存 `Cache-Control: max-age` / `Expires`；协商缓存 `ETag` / `Last-Modified`。  
**`max-age=0, must-revalidate`**：响应可存但立即过期，之后使用前必须向服务器**重新验证**（常带条件请求），不等同于禁止缓存，而是「每次用前要问服务器是否仍有效」。

更多：[浏览器的缓存机制](/md/浏览器/浏览器的缓存机制.md)。

## HTTPS 是如何保证安全的？它的加密过程是怎样的？

**目标**：机密性、完整性、身份认证。  
**TLS 握手（简述）**：协商算法、服务端证书认证（链到可信 CA）、密钥交换生成会话密钥，后续对称加密传输；结合 MAC/AEAD 保证完整性。  
可补充：**证书校验**、**HSTS**、与 HTTP/2、QUIC 的关系（点到为止即可）。

更多：[HTTPS](/md/网络/HTTPS.md)。
