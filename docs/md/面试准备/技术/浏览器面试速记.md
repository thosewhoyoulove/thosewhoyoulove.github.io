# 浏览器与性能速记

## 面试定位

三年前端岗的浏览器题，很少只考 API 名词。面试官更关心：主线程为什么卡、一次点击到页面更新经历了什么、缓存为什么没生效、首屏指标怎么优化和验证。

这类题通常和 **事件循环、渲染流水线、HTTP 缓存、Core Web Vitals、项目性能治理** 绑定。答到「机制 + 项目取舍 + 指标验证」即可，不必堆 V8 字节码细节。

## 面试回答

> 我习惯把浏览器题拆成三条线：执行模型、渲染模型、加载与缓存。
>
> 执行模型上，JS 在渲染进程主线程跑，事件循环先跑同步代码，再清空微任务，再取下一个宏任务；所以 Promise.then 会先于 setTimeout。页面卡顿，本质多是主线程被长任务占满。
>
> 渲染模型上，DOM 和 CSSOM 合成渲染树，再 layout、paint、composite。改 transform/opacity 通常只走合成层，比改 width 触发 layout 更便宜。我会结合 DevTools Performance 看 Long Task 和 Layout Shift。
>
> 加载与缓存上，强缓存看 Cache-Control，协商缓存看 ETag/Last-Modified；SPA 还要区分 HTML 壳、带 hash 的静态资源和 API。性能优化我会先定指标（LCP、INP、CLS），再按网络、渲染、JS 执行、资源体积分层治理，最后用 RUM 或 Lighthouse 回归。

一句话总结：

> 浏览器面试 = 主线程事件循环 + 渲染链路 + 缓存策略 + 可量化性能指标。

## 核心原理

### 1. 事件循环（必考）

| 类型 | 常见来源 | 执行时机 |
| --- | --- | --- |
| 宏任务 | script、setTimeout、I/O、UI 事件 | 每轮取一个 |
| 微任务 | Promise.then、queueMicrotask、MutationObserver | 当前宏任务结束后清空 |

记忆口诀：**同步 → 微任务 → （可能渲染）→ 下一宏任务**。

三年岗常考输出题：`async/await` 本质是 Promise，仍走微任务；`await` 后的代码相当于 `.then`。

### 2. 渲染流水线（高频）

```text
DOM + CSSOM → Render Tree → Layout → Paint → Composite
```

| 操作 | 典型触发 | 面试表达 |
| --- | --- | --- |
| 重排（reflow/layout） | 改几何属性（width、font-size） | 代价高，尽量批量读写 |
| 重绘（repaint） | 改颜色、visibility | 比 layout 轻，仍占主线程 |
| 合成（composite） | transform、opacity、will-change | 优先用合成属性做动画 |

### 3. 缓存（线上问题常追）

| 类型 | 关键头 | 特点 |
| --- | --- | --- |
| 强缓存 | Cache-Control / Expires | 未过期不发请求 |
| 协商缓存 | ETag / Last-Modified | 304 复用本地副本 |

三年岗要能讲清：**HTML 是否缓存、带 contenthash 的 JS/CSS 如何长缓存、API 为什么不走强缓存**。

### 4. 三年岗性能指标（2024 起更常问 INP）

| 指标 | 含义 | 优化方向 |
| --- | --- | --- |
| LCP | 最大内容绘制 | 关键资源优先级、图片格式、SSR/预加载 |
| INP | 交互到下一帧 | 减少长任务、拆包、Worker、防抖 |
| CLS | 布局偏移 | 预留尺寸、字体 fallback、避免动态插入 |

### 5. 加分项（点到为止）

- 多进程架构：渲染、网络、GPU 隔离，单 tab 崩溃不拖垮浏览器。
- Web Worker：把计算密集任务移出主线程，注意不能操作 DOM。
- 高阶细节见 [浏览器高阶知识点](/md/浏览器/高阶知识点.md)。

## 高频追问

### 为什么 setTimeout(fn, 0) 不是立刻执行？

因为它进宏任务队列，要等当前同步代码和所有微任务跑完。

### 首屏优化你会先做哪三件事？

先量 LCP/FCP 找瓶颈，再压关键路径（路由级分包、预加载、图片/CDN），最后验证缓存与长任务（Performance + 线上 RUM）。

### 强缓存和协商缓存同时配怎么理解？

浏览器先查强缓存，过期再走协商；带 hash 的静态资源通常强缓存一年，HTML 往往 no-cache 保证发版。

### Web Worker 和 Service Worker 区别？

Worker 跑脚本不占 UI 线程；Service Worker 是网络代理层，管离线缓存和 Push，生命周期和页面不同。
