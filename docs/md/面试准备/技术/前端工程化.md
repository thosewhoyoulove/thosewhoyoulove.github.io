# 前端工程化

## 面试定位

前端工程化考察的是你能不能把个人开发能力扩展成团队交付能力。

面试官通常会追问：构建工具为什么这样选、如何优化构建速度、如何保证代码质量、CI/CD 怎么设计、Monorepo 是否适合当前团队。

## 核心原理

前端工程化的本质是用工具、规范和流程降低复杂度。

它通常覆盖这些部分：

- 开发效率：脚手架、热更新、模块化、类型检查。
- 构建效率：缓存、并行、按需编译、依赖预构建。
- 代码质量：Lint、Format、TypeScript、测试、Code Review。
- 发布稳定性：CI/CD、灰度、回滚、监控。
- 协作一致性：目录规范、提交规范、文档、组件库、Monorepo。

工程化不是堆工具，而是让团队在多人协作和长期维护下还能稳定交付。

## Webpack 和 Vite 怎么对比

Webpack 是传统打包器，开发时通常需要先构建依赖图，再把模块打包后提供给浏览器。

Vite 在开发态利用浏览器原生 ESM，不需要一开始把整个项目打包。源码按需编译，第三方依赖用 esbuild 预构建，所以冷启动和热更新更快。

可以这样回答：

```text
Webpack 更像先整体分析和打包，再交给浏览器运行。
Vite 开发态更像按需提供模块，浏览器请求哪个模块才处理哪个模块。
生产环境 Vite 仍然会基于 Rollup 做打包优化。
```

## 常见构建优化

Webpack 常见优化：

- 缩小 loader 处理范围。
- 开启持久化缓存。
- 使用 `thread-loader` 或 esbuild 加速转译。
- 合理配置 `splitChunks`。
- 开启 Tree Shaking。
- 分析包体积，移除无用依赖。

Vite 常见优化：

- 合理配置依赖预构建。
- 避免过大的 CommonJS 依赖影响转换。
- 用动态 import 做路由和组件分包。
- 对大型依赖做 CDN external 或按需引入。

## Tree Shaking 原理

Tree Shaking 是删除未使用代码。

它依赖 ES Module 的静态结构，因为 `import` 和 `export` 在编译阶段就能分析出依赖关系。

生效条件包括：

- 使用 ESM。
- 避免模块有不可分析的副作用。
- 正确配置 `sideEffects`。
- 不要把所有模块都打成难以分析的 CommonJS。

面试里可以补一句：Tree Shaking 不等于压缩，压缩是缩短代码，Tree Shaking 是删除没有被引用的代码。

## CI/CD 怎么设计

一个常见流程是：

```text
提交代码 -> 静态检查 -> 单元测试 -> 构建 -> 生成预览环境 -> 合并主干 -> 部署 -> 健康检查 -> 监控和回滚
```

核心不是“用了某个平台”，而是每个阶段解决什么风险。

- Lint 解决规范问题。
- TypeScript 解决类型问题。
- Test 解决逻辑回归。
- Build 解决产物可用性。
- Preview 解决验收问题。
- Monitor 和 rollback 解决上线风险。

## Monorepo 适合什么场景

Monorepo 适合多个项目共享组件、工具、类型、配置，且这些包需要频繁协同变更的场景。

优点：

- 共享代码更方便。
- 统一规范和配置。
- 跨包改动可以一次提交。
- 配合缓存和增量构建提升效率。

代价：

- 仓库更大。
- 权限和发布更复杂。
- CI 需要做影响范围分析。
- 包边界不清时容易互相耦合。

## 跨域怎么处理

跨域是浏览器同源策略导致的问题。

常见方案：

- 开发环境用代理转发。
- 生产环境由服务端配置 CORS。
- 网关或 BFF 做同域转发。
- 老接口可能用 JSONP，但现代项目很少推荐。

如果涉及 Cookie，需要额外注意 `Access-Control-Allow-Credentials`、`SameSite` 和具体域名配置，不能简单写 `*`。

## 面试回答

我理解的前端工程化不是单纯配置 Webpack 或 Vite，而是围绕效率、质量和稳定性交付建立体系。

开发阶段要解决启动慢、热更新慢、类型和规范不统一的问题，所以会用 Vite、TypeScript、ESLint、Prettier、统一脚手架这些工具。构建阶段关注包体积、分包、缓存、Tree Shaking 和构建速度。协作阶段要有 Code Review、提交规范和文档沉淀。上线阶段则通过 CI/CD、测试、预览环境、灰度、监控和回滚来降低风险。

如果具体讲 Vite 和 Webpack，我会说 Vite 开发态利用原生 ESM 和依赖预构建，所以启动和热更新更快；Webpack 更成熟，生态和复杂定制能力强。选型时不是看谁绝对更好，而是看项目规模、历史包袱、团队熟悉度和构建定制需求。

## 高频追问

### Vite 为什么快？

开发态不需要一开始打包整个项目，而是利用浏览器原生 ESM 按需加载源码。

第三方依赖会用 esbuild 预构建，源码修改时只需要更新相关模块，所以冷启动和热更新都更快。

### loader 和 plugin 的区别是什么？

loader 主要负责转换某类文件，比如把 TS 转成 JS，把 Less 转成 CSS。

plugin 能介入构建生命周期，做更全局的事情，比如生成 HTML、压缩资源、分析包体积、注入环境变量。

### CI/CD 如何保证质量？

通过流水线把检查前置。

常见包括 Lint、类型检查、单元测试、构建校验、预览环境、自动部署、健康检查和回滚策略。

### Monorepo 会带来什么问题？

最大问题是复杂度上升。

需要做好包边界、版本管理、增量构建、权限控制和 CI 缓存，否则仓库会变慢，包之间也容易耦合。
