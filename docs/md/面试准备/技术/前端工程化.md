## 你在项目中用 Webpack 和 Vite 进行过哪些优化？为什么 Vite 比 Webpack 更快？

**Webpack 常见优化**：代码分割、`splitChunks`、持久化缓存、`thread-loader`/`esbuild-loader`、合理 `resolve`、缩小 `babel` 范围、`tree-shaking`、压缩与 CDN。  
**Vite 快的原因（开发态）**：利用浏览器原生 ESM、依赖预构建（esbuild）、按需编译当前模块；生产态仍多使用 Rollup 打包，与开发体验分离。  
对比表述：Webpack 偏「先打包再服务」，Vite 开发态偏「按需编译 + 预构建依赖」。

## Tree Shaking 是如何工作的？它的原理是什么？

**原理**：基于 ES Module **静态结构**（`import`/`export` 在顶层、可静态分析），打包器做 DCE（死代码消除），未引用的导出不进最终 bundle。  
**生效条件**：副作用标注（`sideEffects` 字段）、避免把整个库打成 CJS 导致难以摇树、注意 re-export 与动态 import 边界。

## 你提到在项目中用过 Monorepo，你是怎么管理多个包的？

**思路**：pnpm/npm workspace、Turborepo/Nx 任务编排与缓存、共享 tsconfig/eslint、版本发布（Changesets）、私有包依赖链接、CI 分层只构建受影响包。详见 [Monorepo](/md/工程化/体系与实践/Monorepo.md)。

## CI/CD 你是怎么做的？从提交代码到上线的完整流程？

**通用流程**：push/MR → **lint + unit test + build** → 预览环境（可选）→ merge 主干 → **自动打 tag/版本** → 构建产物 → 部署（静态托管/K8s/Serverless）→ 健康检查与回滚策略。  
可提：缓存依赖、并行 job、环境变量与密钥管理、灰度与开关。

## 你在项目中如何处理跨域问题？CORS、JSONP、代理服务器？

**CORS**：服务端 `Access-Control-Allow-Origin` 等响应头，预检请求见 [OPTIONS 预检](/md/网络/OPTIONS预检请求.md)。  
**开发代理**：Vite/Webpack devServer `proxy` 同源转发，避免浏览器跨域。  
**JSONP**：仅 GET、仅适合老式接口，有安全风险，现代项目少用。  
生产环境优先：**网关同域**、BFF、或正确配置 CORS + 凭证策略。
