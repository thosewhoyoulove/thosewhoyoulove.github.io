# 前端工程化速记

## 面试定位

前端工程化考察的是你能不能把个人开发能力扩展成团队交付能力。复习时不要把它背成工具清单，要围绕效率、质量、构建、发布、协作来回答。

## 面试回答

> 我理解的前端工程化不是单纯配置 Webpack 或 Vite，而是围绕团队长期稳定交付建立工具、规范和流程。开发阶段要解决启动慢、热更新慢、类型和规范不统一的问题；构建阶段关注打包速度、包体积、分包、Tree Shaking、Source Map 和缓存；质量阶段通过 ESLint、TypeScript、测试、Code Review 和 CI 把问题前置；发布阶段要有预览环境、灰度、监控和回滚；协作阶段要有目录规范、提交规范、组件库、文档和 Monorepo 边界治理。选型时不是看工具谁更流行，而是看项目规模、历史包袱、团队熟悉度和定制需求。

一句话总结：

> 前端工程化是用工具、规范和流程降低多人协作、长期维护和稳定发布的复杂度。

## 高频追问

### Webpack 和 Vite 怎么对比？

Webpack 开发态通常要先构建依赖图并打包，生态成熟、定制能力强；Vite 开发态利用浏览器原生 ESM 按需编译，冷启动和 HMR 更快。生产环境 Vite 仍然使用 Rollup 打包。

### loader 和 plugin 的区别是什么？

loader 处理单个模块，把 TS、CSS、图片等转换成 Webpack 可处理的模块；plugin 通过生命周期钩子介入整个构建流程，能生成资源、压缩、分析包体积或上传产物。

### 构建优化怎么证明有效？

用指标说话：构建耗时、缓存命中、主包体积、请求瀑布图、LCP、路由切换耗时和线上错误定位成功率。

### CI/CD 如何保证质量？

把检查前置到流水线：lint、类型检查、测试、构建、预览环境、部署、健康检查、监控和回滚。
