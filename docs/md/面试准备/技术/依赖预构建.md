# 依赖预构建速记

## 面试定位

这页只保留速记入口，完整原理以 [Vite 依赖预构建](/md/工程化/Vite/依赖预构建.md) 为准，避免同一知识点双份维护。

## 面试回答

> Vite 依赖预构建主要服务开发环境。因为浏览器原生 ESM 不能直接处理很多 CommonJS/UMD 依赖，而且有些 ESM 依赖内部模块非常碎，浏览器逐个请求会很慢。Vite 会用 esbuild 把第三方依赖提前转成 ESM，并合并成更少的模块，缓存到 `node_modules/.vite`，从而提升启动和页面加载速度。生产构建仍然主要由 Rollup 处理，不能把依赖预构建等同于线上分包。

一句话总结：

> 依赖预构建 = 开发态用 esbuild 把第三方依赖转成更适合浏览器加载的 ESM 缓存。
