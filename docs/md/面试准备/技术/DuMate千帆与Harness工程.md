# DuMate、千帆与 Harness 工程

## 面试定位

这是 **DuMate-全栈研发** 岗位的业务与技术交叉题。面试官想确认：你知道产品在做什么、Harness 不是泛泛的「工程化口号」，以及你能否把 **Agent 稳定性、成本、安全、体验** 和研发工作联系起来。

不要求背百度内部架构细节，但要能讲清 **公开产品定位 + Harness 范式 + 你作为研发如何落地**。

## 面试回答

可以这样答：

> DuMate 是百度推出的通用智能体产品，面向个人和团队办公场景，能看屏幕、操作软件、处理文件、串联业务系统，从想法到结果尽量自动化，手机和 PC 可协同，也是接入搜索、秒哒、伐谋等能力的统一入口之一，依托千帆等智能云基础设施。
>
> Harness 工程（驾驭工程）是百度在 Agent 时代强调的 AI 工程化范式：大模型能力强但不可控，需要像「马具」一样给它规则约束、工具链路（例如 MCP）、运行环境（如安全沙箱）、观测与反馈闭环，提高任务成功率，并控制 Token 消耗。公开报道里 DuMate 配合 Harness 在办公类任务上成功率更高、Token 更省，并有本地沙箱、授权访问、高危拦截等安全设计。
>
> 我作为全栈研发，前端会关注对话与任务状态、流式反馈、工具执行结果展示、失败可恢复和用户体验；协作上会做好 BFF 或接口聚合、鉴权与错误码统一；工程上会配合 prompt/工具版本、日志 trace、评测与灰度，让迭代可验证，而不是只堆模型能力。

一句话总结：

> DuMate = 千帆体系下的通用 Agent 产品；Harness = 让 Agent 可控、可迭代、可省成本的工程范式；全栈研发 = 体验 + 接口 + 稳定性 + 可观测闭环。

## 核心原理

### 1. DuMate 是什么

（基于 [百度智能云 DuMate 文档](https://cloud.baidu.com/doc/Dumate/s/xmmu7qrvo) 与公开发布会信息，面试以「产品能力」表述即可。）

| 维度 | 要点 |
| --- | --- |
| 定位 | 桌面/移动端 **通用智能体（AI 搭子）**，办公提效 |
| 能力 | 屏幕理解、软件操作、文件处理、跨系统业务串联 |
| 体验 | 长程任务、主动决策、用户最终审核确认 |
| 生态 | 与千帆、搜索 API、秒哒、伐谋等能力集成 |
| 安全 | 本地沙箱、数据本地流转、授权访问、高危操作拦截 |

和「纯聊天大模型」的差异：**强调执行与交付结果**，而不只是生成文本。

### 2. 千帆与岗位关系

- **千帆**：百度智能云大模型与 Agent 开发平台（模型、工具、企业级能力）。
- **千帆公有云前端组**：做千帆相关产品的 **Web 端体验与全栈交付**；DuMate 是其中一条产品线。
- 面试可表达：愿意做 **面向企业/开发者的 AI 产品**，而不只做传统 CRUD 后台。

### 3. Harness 工程（驾驭工程）

公开语境下的核心逻辑（非单一开源库名，而是 **方法论**）：

```text
模型能力（野马）
  → Harness：规则 / 工具(MCP) / 沙箱 / 观测
  → 任务规划与执行
  → 反馈与评测（成功率、Token、时延）
  → 持续优化 prompt、工具、策略
```

| 要素 | 作用 |
| --- | --- |
| 规则与约束 | 减少幻觉与越权操作 |
| 工具链路 | 调 Word/浏览器/业务 API，结构化 I/O |
| 运行环境 | 沙箱隔离、权限、高危拦截 |
| 反馈闭环 | 失败重试、人工确认、日志与评测 |
| Token 治理 | 缓存、裁剪上下文、选型与路由 |

和「平时工作怎么使用 AI」的区别：Harness 是 **产品级/平台级** 把 AI 嵌进工程体系；个人用 AI 是研发提效。二者可结合讲：你写代码也用 AI，但交付 Agent 功能时要 **可测、可回滚、可观测**。

### 4. 全栈研发在本岗位的映射

| JD 条目 | 可答要点 |
| --- | --- |
| 千帆相关产品全栈研发 | React 界面 + Node BFF + 与 Go 服务协作 |
| 研究 Harness，优化实现 | 工具注册、执行状态机、评测指标、版本灰度 |
| 快速稳定迭代 | CI/CD、feature flag、监控告警、回滚 |
| 用户视角提升体验 | 任务进度、可中断、错误可理解、结果可审核 |
| 端到端落地 | 需求 → 方案 → 前后端 → 联调 → 指标 |

## 高频追问

### Harness 和 Prompt Engineering 有什么区别？

Prompt 是一次性「怎么说」；Harness 是 **长期运行的系统**：规则、工具、环境、监控、评测一起保证 Agent **重复任务也稳定**，并能量化优化。

### 前端在 DuMate 类项目里最重要什么？

**任务与消息状态机**、流式输出、工具调用过程可视化、失败重试与人工确认、性能（长列表/增量 Markdown）、安全提示（授权与敏感操作）。

### 如何体现 product sense？

说清 **用户场景 → 为什么自动化 → 成功标准（省时/成功率）→ 你怎么验证**；避免只列技术栈。

### Token 成本前端能做什么？

展示上下文用量、引导用户精简输入、避免重复上传大附件；配合后端做会话摘要、缓存检索结果（概念即可）。
