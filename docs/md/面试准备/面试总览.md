## 答题原则

面试回答尽量遵循这个结构：

1. 先给一句结论。
2. 再解释核心原理。
3. 补一个项目场景。
4. 主动说边界和取舍。
5. 最后给验证方式或指标。

例如问“怎么做首屏优化”，不要只背清单。更好的回答是：先说目标指标，再说资源加载、渲染链路、代码分包、缓存策略、监控验证，最后结合自己的项目说明哪项收益最大。
