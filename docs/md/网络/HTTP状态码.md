# HTTP 状态码

## 面试定位

状态码是网络基础题，也常用于项目排障。面试回答不要只背数字，要能说清状态码类别、常见码含义、前后端分别该如何处理。

## 核心原理

HTTP 状态码是服务器对请求处理结果的标准化表达。

分类：

- `1xx`：信息提示，请求仍在处理。
- `2xx`：成功。
- `3xx`：重定向或缓存相关。
- `4xx`：客户端请求有问题。
- `5xx`：服务端处理有问题。

## 常见 2xx

### 200 OK

请求成功，响应体里通常有数据。

### 201 Created

资源创建成功，常用于 POST 创建资源。

### 204 No Content

请求成功，但没有响应体。常用于 DELETE 或某些更新操作。

### 206 Partial Content

范围请求成功，常用于视频拖拽、断点续传。

## 常见 3xx

### 301 Moved Permanently

永久重定向。浏览器和搜索引擎可能缓存这个重定向结果。

### 302 Found

临时重定向。资源临时移动到其他位置。

### 304 Not Modified

协商缓存命中。服务器告诉浏览器资源没变，浏览器继续使用本地缓存。

```http
HTTP/1.1 304 Not Modified
ETag: "abc123"
```

### 307 / 308

307 是临时重定向，308 是永久重定向。它们会保留原请求方法和请求体，比 301/302 在语义上更严格。

## 常见 4xx

### 400 Bad Request

请求格式或参数错误。

### 401 Unauthorized

未认证或认证失效。常见处理是跳转登录或刷新 token。

### 403 Forbidden

已认证，但没有权限访问。

### 404 Not Found

资源不存在。

### 405 Method Not Allowed

请求方法不被允许。

### 409 Conflict

资源冲突，例如重复创建、版本冲突。

### 413 Payload Too Large

请求体过大，常见于上传文件超过限制。

### 429 Too Many Requests

请求过于频繁，被限流。

## 常见 5xx

### 500 Internal Server Error

服务器内部错误，通常是代码异常或未处理错误。

### 502 Bad Gateway

网关或代理从上游服务收到无效响应。常见于 Nginx 代理后端服务异常。

### 503 Service Unavailable

服务暂时不可用，可能是维护、过载、限流。

### 504 Gateway Timeout

网关等待上游服务超时。

## 面试回答

可以这样答：

> HTTP 状态码表示服务器对请求的处理结果。`2xx` 表示成功，比如 200 成功、201 创建成功、204 成功但无响应体；`3xx` 表示重定向或缓存，比如 301 永久重定向、302 临时重定向、304 协商缓存命中；`4xx` 表示客户端侧问题，比如 400 参数错误、401 未登录、403 无权限、404 资源不存在、429 请求过多；`5xx` 表示服务端问题，比如 500 服务异常、502 网关收到异常响应、503 服务不可用、504 网关超时。实际项目排查时，我会结合状态码、请求参数、响应体、网关日志和服务日志定位问题。

## 高频追问

### 401 和 403 区别？

401 是未认证或认证失效，用户身份还没确认。403 是身份已确认，但没有权限。

### 301 和 302 区别？

301 是永久重定向，适合域名迁移、路径永久变更；302 是临时重定向，适合临时跳转。301 可能被浏览器和搜索引擎缓存，使用要谨慎。

### 502 和 504 区别？

502 是网关从上游拿到了无效响应或上游异常；504 是网关等待上游响应超时。

### 304 是错误吗？

不是。304 表示协商缓存命中，服务器没有返回响应体，浏览器继续使用本地缓存。
