# 浏览器高阶知识点

## 面试定位

这篇不是用来背诵所有细节，而是用于面试加分。基础题答稳后，可以结合项目点到为止地补充 1 到 2 个高阶方向，体现你理解浏览器不止停留在 API 使用。

## 核心原理

浏览器不是单一的 JavaScript 执行器，而是一套由进程隔离、渲染流水线、网络栈、安全沙箱、存储策略和 JavaScript 引擎共同组成的运行环境。高阶题的回答重点不是堆名词，而是把这些能力和性能、安全、稳定性、用户体验联系起来。

## 浏览器多进程架构

现代浏览器通常是多进程架构，常见进程包括：

- Browser 进程：负责地址栏、书签、进程管理、权限等。
- Renderer 进程：负责页面解析、JS 执行、布局、绘制等。
- Network 进程：负责网络请求。
- GPU 进程：负责图形合成和部分渲染加速。
- 插件或扩展进程：隔离插件和扩展风险。

多进程带来的好处：

- 稳定性更好，单个页面崩溃不一定影响整个浏览器。
- 安全性更好，不同站点可以隔离。
- 性能更可控，渲染、网络、GPU 可以分工。

面试表达：

> 浏览器不是一个单线程程序。JavaScript 在渲染进程的主线程执行，但网络、GPU 合成等能力可能在其他进程或线程中完成。页面卡顿通常说的是渲染主线程被阻塞。

## 站点隔离和浏览器沙箱

站点隔离是把不同站点尽量放到不同进程中，降低跨站数据泄露风险。浏览器沙箱则限制渲染进程直接访问系统资源，即使页面执行了恶意代码，也不能轻易越权操作系统。

面试中不需要深入操作系统细节，可以说清目标：隔离风险、降低攻击面。

## V8 执行流程

V8 执行 JavaScript 大致可以理解为：

1. 解析源码，生成 AST。
2. 生成字节码。
3. 解释器执行字节码。
4. 热点代码交给优化编译器优化。
5. 如果假设失效，可能发生反优化。

面试回答不要陷入细枝末节，重点说：现代 JS 引擎不是简单逐行解释，它会解释执行和即时优化结合。

## 资源提示

资源提示可以优化关键资源加载。

```html
<link rel="dns-prefetch" href="//cdn.example.com" />
<link rel="preconnect" href="https://cdn.example.com" />
<link rel="preload" href="/main.css" as="style" />
<link rel="prefetch" href="/next-page.js" as="script" />
```

区别：

- `dns-prefetch`：提前 DNS 解析。
- `preconnect`：提前建立连接。
- `preload`：当前页面马上需要的高优先级资源。
- `prefetch`：未来可能需要的低优先级资源。

使用原则：只对关键资源使用，不要滥用，否则会抢占带宽。

## HTTP/3 和 QUIC

HTTP/3 基于 QUIC，QUIC 基于 UDP 实现可靠传输。

可以记住几个面试点：

- 改善 TCP 层队头阻塞。
- 支持连接迁移，例如移动网络切换。
- 握手延迟更低，支持 0-RTT，但 0-RTT 有重放风险。

不要把 HTTP/3 简单说成“UDP 不可靠所以更快”。QUIC 是在 UDP 之上实现可靠传输、拥塞控制和加密。

## 存储与隐私趋势

浏览器对第三方 Cookie、跨站跟踪和嵌入式 iframe 的限制越来越严格。

常见点：

- `SameSite` Cookie 限制跨站发送。
- 第三方 Cookie 逐步受限。
- 分区存储让第三方上下文的数据隔离。
- Storage Access API 用于某些嵌入场景申请访问权限。

这类问题适合在登录态、SSO、嵌入页、广告统计等项目里提及。

## 面试回答

可以这样答：

> 浏览器高阶能力我一般会结合项目场景回答。比如性能问题不只看 JS，还要知道浏览器是多进程架构，JS 主要阻塞的是渲染主线程；首屏优化可以用 preload、preconnect 等资源提示，但要避免滥用；安全上浏览器有站点隔离和沙箱来降低跨站和恶意页面风险；网络上 HTTP/3 基于 QUIC，改善了 TCP 队头阻塞和连接迁移问题。面试里我不会空泛展开，而是把这些点和项目里的性能优化、安全策略或网络链路结合起来。

## 高频追问

### 浏览器多进程和 JavaScript 单线程矛盾吗？

不矛盾。浏览器整体是多进程、多线程协作，但单个页面里的 JavaScript 通常在渲染主线程上单线程执行。

### preload 和 prefetch 区别？

`preload` 用于当前页面马上需要的关键资源，优先级高。`prefetch` 用于未来可能访问的资源，优先级低。

### HTTP/3 为什么能改善队头阻塞？

TCP 层丢包会影响同一连接上的所有 HTTP/2 流。QUIC 在传输层支持多路流，单个流丢包不会阻塞其他流的数据交付。
