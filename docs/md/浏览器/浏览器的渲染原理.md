# 浏览器的渲染原理

## 面试定位

渲染原理常用于考察性能优化基础。面试官通常会追问：从 HTML 到页面展示发生了什么、CSS 和 JS 为什么会阻塞、重排和重绘区别、如何避免布局抖动、为什么动画推荐用 `transform`。

## 核心原理

浏览器把 HTML、CSS、JavaScript 和资源转换成屏幕像素，大致经历：

1. 解析 HTML，构建 DOM。
2. 解析 CSS，构建 CSSOM。
3. DOM 和 CSSOM 合成渲染树。
4. Layout，计算元素位置和尺寸。
5. Paint，生成绘制指令。
6. Composite，合成图层并显示到屏幕。

## DOM 和 CSSOM

HTML 解析会生成 DOM 树。

```html
<div class="card">
    <h1>Hello</h1>
</div>
```

CSS 解析会生成 CSSOM，包含选择器、继承、层叠和最终计算样式。

```css
.card {
    width: 200px;
    color: red;
}
```

DOM 描述结构，CSSOM 描述样式。两者结合后，浏览器才知道哪些节点要展示，以及展示成什么样。

## CSS 和 JS 的阻塞

CSS 通常会阻塞渲染。浏览器需要 CSSOM 才能计算最终样式，如果 CSS 没加载完就绘制，可能出现无样式闪烁。

普通 `<script>` 会阻塞 HTML 解析。因为脚本可能修改 DOM 或样式，浏览器必须先执行脚本再继续解析。

```html
<script src="/main.js"></script>
```

优化方式：

- 把非关键脚本加 `defer`。
- 第三方脚本谨慎使用 `async`。
- 提取关键 CSS。
- 减少首屏阻塞资源体积。

```html
<script src="/main.js" defer></script>
```

`defer` 会并行下载脚本，等 HTML 解析完成后按顺序执行。`async` 会并行下载，下载完成后立即执行，不保证顺序。

## 渲染树

渲染树只包含需要展示的节点。`display: none` 的元素不会进入渲染树，`visibility: hidden` 的元素仍占据布局空间。

这也是两者性能和布局表现不同的原因。

## Layout：布局

Layout 也叫 Reflow，会计算元素的几何信息，例如宽、高、位置。

会触发布局的常见变化：

- 修改宽高、边距、位置、字体大小。
- 新增或删除 DOM。
- 改变窗口大小。
- 读取某些布局属性前有未完成的样式修改。

常见布局读取属性：

- `offsetWidth`
- `offsetHeight`
- `clientWidth`
- `getBoundingClientRect()`

如果在循环里频繁写样式再读布局，就可能触发强制同步布局。

```javascript
// 不推荐：读写交错
items.forEach((item) => {
    item.style.width = "100px";
    console.log(item.offsetWidth);
});
```

更好的方式是批量读、批量写。

## Paint：绘制

Paint 会把元素的颜色、文字、边框、阴影、背景等转换成绘制指令。

只改变视觉效果、不改变几何信息的属性，通常可能只触发重绘，比如：

- `color`
- `background-color`
- `box-shadow`

但具体代价仍然和影响范围、图层、浏览器优化有关。

## Composite：合成

浏览器会把页面分成多个图层，最后由合成线程把图层合成为最终画面。

`transform` 和 `opacity` 经常可以只触发合成，不必重新布局或大面积重绘，所以动画里常推荐使用它们。

```css
.box {
    transform: translateX(100px);
    opacity: 0.8;
}
```

但图层不是越多越好。过度使用 `will-change` 会增加内存和合成成本。

## 重排和重绘

重排：元素几何信息变化，需要重新计算布局。通常成本较高，并且可能引起后续重绘和合成。

重绘：元素外观变化，但几何信息不变，只需要重新绘制。

合成：只改变已有图层的组合方式，例如 `transform`、`opacity` 动画，通常成本较低。

优化原则：

- 减少 DOM 层级和大范围样式影响。
- 避免频繁读写布局属性。
- 批量 DOM 操作。
- 使用 `requestAnimationFrame` 处理动画。
- 长列表使用虚拟列表。
- 动画优先使用 `transform` 和 `opacity`。

## 面试回答

可以这样答：

> 浏览器渲染大致分为解析 HTML 生成 DOM、解析 CSS 生成 CSSOM、合成渲染树、布局、绘制和合成。CSS 通常会阻塞渲染，因为浏览器需要知道最终样式；普通 script 会阻塞 HTML 解析，因为脚本可能修改 DOM。性能优化里要重点关注重排和重绘。重排是元素几何信息变化，需要重新计算布局，成本较高；重绘是外观变化，不一定重新布局；如果只改变 `transform`、`opacity` 这类合成友好属性，很多时候可以只走合成。实际项目里我会减少阻塞资源、避免读写布局交错、批量 DOM 更新，动画尽量用 `transform`。

## 高频追问

### `display: none` 和 `visibility: hidden` 的区别？

`display: none` 不进入渲染树，不占布局空间。`visibility: hidden` 不可见但仍占据布局空间。

### 为什么读取 `offsetHeight` 可能导致性能问题？

如果之前有样式写入尚未完成布局，读取布局属性会迫使浏览器立即计算最新布局，形成强制同步布局。

### `will-change` 能不能随便用？

不能。它可能提前创建图层，提升某些动画性能，但会增加内存和合成成本，应只用于确实需要优化的元素，并在不需要时移除。
