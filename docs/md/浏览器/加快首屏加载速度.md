# 加快首屏加载速度

## 面试定位

首屏优化是性能面试必考题。面试官通常希望听到完整排查链路，而不是只背“压缩、CDN、懒加载”。回答要围绕指标、资源加载、渲染阻塞、代码执行、缓存和监控闭环。

## 核心原理

首屏慢通常来自四类问题：

1. 网络慢：HTML、JS、CSS、图片等资源下载慢。
2. 阻塞多：CSS、同步 JS、第三方脚本阻塞解析和渲染。
3. 执行慢：主线程被大量 JavaScript、复杂计算或长任务占用。
4. 渲染慢：DOM 复杂、样式计算重、布局抖动、图片尺寸不稳定。

优化前先定义指标：

- FCP：首次内容绘制。
- LCP：最大内容绘制，衡量首屏主体内容出现时间。
- TTI：可交互时间。
- TBT：总阻塞时间。
- CLS：布局偏移。

面试中如果只说“页面变快了”不够，最好说明用什么指标验证。

## 资源加载优化

### 减少首屏资源体积

- 压缩 HTML、CSS、JavaScript。
- 开启 Gzip 或 Brotli。
- Tree shaking 删除无用代码。
- 拆分路由级代码，首屏只加载必要模块。
- 移除无用 polyfill 和重复依赖。

### 使用 CDN

静态资源放 CDN 可以降低网络延迟，提高并发下载能力。需要配合合理缓存策略。

### 图片优化

- 使用 WebP、AVIF 等更小格式。
- 按展示尺寸裁剪图片，避免加载超大图。
- 图片懒加载。
- 给图片设置宽高，减少布局偏移。
- 首屏关键图可以预加载。

```html
<img src="banner.webp" width="1200" height="600" loading="eager" />
<img src="list.webp" loading="lazy" />
```

## 渲染阻塞优化

### CSS

CSS 会阻塞渲染，首屏关键 CSS 应尽量小。

常见做法：

- 提取关键 CSS。
- 延迟加载非首屏 CSS。
- 避免过深、过复杂选择器。

### JavaScript

普通脚本会阻塞 HTML 解析。

```html
<script src="/main.js" defer></script>
```

`defer` 适合大多数业务脚本：并行下载，HTML 解析完成后按顺序执行。

`async` 适合独立第三方脚本：并行下载，下载完立即执行，不保证顺序。

## 代码执行优化

JavaScript 执行时间过长会阻塞主线程，导致页面无法响应和渲染。

优化方式：

- 拆分长任务。
- 把耗时计算放到 Web Worker。
- 减少首屏同步初始化逻辑。
- 延迟非关键逻辑，比如埋点、推荐、非首屏组件。
- 长列表使用虚拟列表。

```javascript
requestIdleCallback(() => {
    // 非关键任务
});
```

`requestIdleCallback` 适合低优先级任务，但不能依赖它执行关键逻辑。

## 缓存优化

HTML 入口通常不强缓存，保证能拿到最新资源引用。

```http
Cache-Control: no-cache
```

带 hash 的 JS、CSS、图片可以长期缓存。

```http
Cache-Control: public, max-age=31536000, immutable
```

这样二次访问速度会明显提升。

## 预加载和预连接

资源提示可以缩短关键资源等待时间。

```html
<link rel="preconnect" href="https://cdn.example.com" />
<link rel="preload" href="/font.woff2" as="font" type="font/woff2" crossorigin />
```

使用原则：

- `preload` 只给首屏关键资源。
- `prefetch` 用于未来可能访问的资源。
- `preconnect` 用于提前建立第三方连接。
- 不要滥用，过多资源提示会抢占带宽。

## 面试回答

可以这样答：

> 首屏优化我会先看指标，比如 FCP、LCP、TBT、CLS，而不是直接改代码。排查方向主要有四类：资源加载、渲染阻塞、主线程执行和缓存。资源层面会压缩资源、开启 Gzip 或 Brotli、使用 CDN、图片用 WebP/AVIF 并按需懒加载；渲染层面会减少阻塞 CSS，脚本用 `defer`，首屏只加载必要代码；执行层面会拆分长任务，非关键逻辑延后，耗时计算放到 Web Worker，长列表用虚拟列表；缓存层面 HTML 用 `no-cache`，带 hash 的静态资源长期缓存。最后通过 Lighthouse、Performance 面板或线上监控对比优化前后的指标。

## 高频追问

### `defer` 和 `async` 区别？

两者都会并行下载脚本。`defer` 等 HTML 解析完成后按顺序执行，适合业务脚本。`async` 下载完立即执行，不保证顺序，适合独立第三方脚本。

### LCP 慢通常怎么优化？

先找 LCP 元素。如果是图片，优化图片格式、尺寸、CDN、预加载；如果是文本或组件，减少阻塞 CSS/JS，降低首屏包体积和主线程长任务。

### 为什么图片要设置宽高？

浏览器可以提前预留布局空间，避免图片加载完成后撑开页面导致 CLS。
