# Vue 模板编译流程

## 面试定位

模板编译是 Vue 区别于纯 JSX 运行时框架的重要能力。面试重点是 template 如何变成 render 函数，以及 Vue 3 如何通过编译优化减少运行时 diff 成本。

## 面试回答

> Vue 模板不会直接变成真实 DOM，而是先经过编译器生成 render 函数。编译大致分三步：parse 把 template 解析成 AST；transform 在 AST 上识别指令、表达式、动态节点，并做静态提升、patchFlag、事件缓存、Block Tree 等优化；codegen 把优化后的 AST 生成 render 函数代码。运行时执行 render 函数得到 VNode，再由 patch 把 VNode 转成真实 DOM 或更新 DOM。Vue 的优势是模板提供了静态分析空间，编译器能提前知道哪些节点是静态的、哪些属性是动态的，更新时就能跳过大量不变内容。

一句话总结：

> Vue 模板编译 = template -> AST -> transform 优化 -> render 函数 -> VNode -> patch。

## 核心原理

```text
template
  -> parse：解析成 AST
  -> transform：处理指令、表达式、静态提升、patchFlag
  -> codegen：生成 render 函数
  -> render 执行生成 VNode
  -> patch 更新真实 DOM
```

示意：

```vue
<template>
  <div>
    <p>{{ msg }}</p>
  </div>
</template>
```

编译后可以理解为：

```js
function render(ctx) {
  return h("div", null, [h("p", null, ctx.msg)]);
}
```

真实 Vue 3 编译结果会包含更多优化信息，比如动态文本标记、静态节点提升等。

## 高频追问

### Vue 为什么能做编译优化？

模板语法比任意 JavaScript 更受约束，编译器可以静态分析节点结构、动态绑定、指令和事件，从而提前标记动态部分。

### patchFlag 是什么？

patchFlag 是编译阶段生成的动态标记，告诉运行时这个 VNode 哪些部分可能变化，比如文本、class、style、props。更新时可以更精准地 patch。

### 静态提升解决什么问题？

不会变化的节点或对象可以提升到 render 函数外部，避免每次组件更新都重新创建，减少运行时开销。
