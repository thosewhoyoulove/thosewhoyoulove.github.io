# Node.js 的事件循环

## 面试定位

Node.js 事件循环常用于考察后端基础和异步机制。回答时要区分浏览器事件循环和 Node.js 事件循环，重点讲阶段、微任务、`process.nextTick`、`setTimeout` 和 `setImmediate`。

## 核心原理

Node.js 基于 V8 执行 JavaScript，基于 libuv 处理异步 I/O、定时器、线程池和事件循环。

JavaScript 代码仍然在主线程执行，但文件 I/O、网络 I/O、DNS、部分加密压缩任务可以交给底层系统或线程池处理，完成后把回调放回事件循环。

Node.js 事件循环主要阶段：

1. timers：执行到期的 `setTimeout`、`setInterval`。
2. pending callbacks：执行部分系统层面的延迟回调。
3. idle / prepare：Node 内部使用。
4. poll：处理 I/O 回调，等待新的 I/O 事件。
5. check：执行 `setImmediate` 回调。
6. close callbacks：执行关闭回调，如 socket close。

## 微任务

Node.js 中常见微任务：

- `process.nextTick`
- Promise microtask
- `queueMicrotask`

需要注意：`process.nextTick` 的优先级通常高于 Promise 微任务。当前同步代码执行完后，会先清空 nextTick 队列，再处理 Promise 微任务，然后进入事件循环阶段。

```javascript
console.log("start");

process.nextTick(() => console.log("nextTick"));
Promise.resolve().then(() => console.log("promise"));

console.log("end");

// start
// end
// nextTick
// promise
```

## setTimeout 和 setImmediate

在主模块中直接执行时，二者先后顺序可能受环境和时机影响。

```javascript
setTimeout(() => console.log("timeout"), 0);
setImmediate(() => console.log("immediate"));
```

但在 I/O 回调中，`setImmediate` 通常先于 `setTimeout`。

```javascript
const fs = require("fs");

fs.readFile(__filename, () => {
    setTimeout(() => console.log("timeout"), 0);
    setImmediate(() => console.log("immediate"));
});

// 通常：immediate -> timeout
```

原因是 I/O 回调在 poll 阶段执行，随后进入 check 阶段执行 `setImmediate`，下一轮 timers 阶段才执行 `setTimeout`。

## 和浏览器事件循环的区别

浏览器事件循环更关注宏任务、微任务和渲染时机；Node.js 没有页面渲染流程，而是围绕 libuv 的多个阶段处理 I/O。

共同点：

- 同步代码先执行。
- Promise 回调属于微任务。
- 长时间同步任务都会阻塞主线程。

差异：

- Node.js 有 `process.nextTick`。
- Node.js 阶段更细，包含 timers、poll、check 等。
- 浏览器还要协调 UI 渲染。

## 面试回答

可以这样答：

> Node.js 的事件循环由 libuv 实现，用来调度定时器、I/O 回调、setImmediate 和关闭回调等任务。它大致分为 timers、pending callbacks、poll、check、close callbacks 等阶段。同步代码先执行，执行完后会先处理微任务，其中 `process.nextTick` 通常优先于 Promise 微任务。`setTimeout` 在 timers 阶段执行，`setImmediate` 在 check 阶段执行。在普通主模块中二者顺序不一定稳定，但在 I/O 回调里通常 `setImmediate` 会先执行。Node 的事件循环和浏览器类似，都是避免异步任务阻塞主线程，但 Node 更关注 I/O 阶段，浏览器还要处理渲染。

## 高频追问

### Node.js 是单线程吗？

JavaScript 执行主线程是单线程的，但 Node.js 底层有 libuv 线程池和操作系统异步 I/O，不是整个运行时只有一个线程。

### process.nextTick 能不能大量使用？

不建议。nextTick 优先级很高，如果递归产生大量 nextTick，可能阻塞事件循环进入 I/O 阶段。

### setImmediate 和 setTimeout(fn, 0) 怎么选？

如果希望在当前 I/O 周期后执行，使用 `setImmediate` 更明确；如果需要基于时间延迟，使用 `setTimeout`。
