# TypeScript 基础类型

## 面试定位

这道题主要考察你是否理解 TypeScript 的类型系统边界：它不是运行时类型检查工具，而是在开发阶段通过静态类型约束降低错误概率。

回答时不要只罗列 `string`、`number`、`boolean`，更要说明 `any`、`unknown`、`never`、联合类型、交叉类型这些类型背后的设计意图。

## 核心原理

TypeScript 的核心是静态类型检查。代码运行前，编译器会根据类型声明和类型推导检查赋值、调用、访问属性等行为是否符合约束。

它的类型系统是结构化类型系统，也就是说 TypeScript 更关注对象“有什么属性和方法”，而不是它来自哪个类或接口。

```ts
type User = {
  id: number;
  name: string;
};

const user = {
  id: 1,
  name: "Tom",
  role: "admin",
};

const currentUser: User = user;
```

上面这段代码可以通过检查，因为 `user` 至少满足 `User` 要求的结构。

## 常见类型

### 基础类型

基础类型包括 `string`、`number`、`boolean`、`null`、`undefined`、`symbol`、`bigint`。

它们用于描述最小粒度的数据形态。

### 联合类型

联合类型表示一个值可以是多个类型之一。

```ts
type ID = string | number;
```

使用联合类型时，只能直接访问所有成员共有的能力。如果要访问某个具体类型的方法，需要先做类型收窄。

### 交叉类型

交叉类型表示把多个类型合并成一个更完整的结构。

```ts
type Base = {
  id: number;
};

type Profile = {
  name: string;
};

type User = Base & Profile;
```

它常用于组合对象能力，例如基础信息、权限信息、业务扩展信息。

## any、unknown、never

### any

`any` 表示放弃类型检查。它可以赋值给任何类型，也可以调用任何属性和方法。

`any` 的风险是把 TypeScript 退回 JavaScript，错误会延后到运行时。

### unknown

`unknown` 表示未知类型。它比 `any` 更安全，因为使用之前必须先判断类型。

```ts
function handle(value: unknown) {
  if (typeof value === "string") {
    return value.toUpperCase();
  }
}
```

面试中可以说：能用 `unknown` 时，不要轻易用 `any`。

### never

`never` 表示永远不会出现的值，常见于抛错函数、死循环、穷尽检查。

```ts
function fail(message: string): never {
  throw new Error(message);
}
```

在联合类型穷尽判断里，`never` 可以帮助发现遗漏分支。

## 面试回答

TypeScript 的类型可以分成几类：基础类型、复合类型和特殊类型。

基础类型用于描述普通值，比如 `string`、`number`、`boolean`。复合类型包括数组、元组、对象、函数、联合类型和交叉类型，用来描述更复杂的数据结构。特殊类型里比较重要的是 `any`、`unknown` 和 `never`。

我一般会避免随意使用 `any`，因为它会绕过类型检查。如果一个值确实来自外部输入，类型暂时不确定，我更倾向于用 `unknown`，然后通过类型判断收窄之后再使用。`never` 则常用于表示不可能出现的情况，比如抛错函数和联合类型的穷尽检查。

TypeScript 的价值不只是标注类型，而是把很多运行时才会暴露的问题提前到开发阶段发现，同时让接口、组件 props、业务数据结构更加清晰。

## 高频追问

### any 和 unknown 有什么区别？

`any` 是放弃类型检查，`unknown` 是保留类型安全。

`any` 可以直接调用任意属性和方法，编译器不会阻止。`unknown` 使用前必须先做类型判断，所以更适合描述外部输入、接口返回、第三方数据等不确定来源。

### type 和 interface 有什么区别？

`interface` 更适合描述对象结构，并且支持同名合并。`type` 能表达更复杂的类型组合，比如联合类型、交叉类型、条件类型。

实际项目里，如果只是定义对象结构，两者都可以；如果涉及类型运算，通常用 `type` 更灵活。

### never 有什么实际用途？

`never` 最常见的用途是做穷尽检查。

当联合类型新增了一个成员，但 `switch` 没有处理到，`never` 会让编译器报错，从而提醒我们补充分支。这在状态机、业务枚举、权限分支里很实用。

### TypeScript 的类型会影响运行时吗？

不会。TypeScript 类型主要在编译阶段生效，最终会被擦除成 JavaScript。

所以它不能替代运行时校验。对于接口返回、表单输入、URL 参数这类外部数据，仍然需要运行时校验。
