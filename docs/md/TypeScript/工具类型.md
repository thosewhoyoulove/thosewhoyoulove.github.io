# TypeScript 工具类型

## 面试定位

这道题主要考察你是否理解 TypeScript 如何基于已有类型派生新类型。

面试里常问 `Partial`、`Pick`、`Omit`、`Record`、`ReturnType` 的用途，也会追问它们背后的 `keyof`、映射类型、条件类型。

## 核心原理

工具类型本质上是 TypeScript 内置的一组类型函数。它们接收一个类型，再返回一个新类型。

比如 `Partial<T>` 会把对象类型的所有属性变成可选。

```ts
type User = {
  id: number;
  name: string;
  age: number;
};

type UpdateUser = Partial<User>;
```

`UpdateUser` 等价于：

```ts
type UpdateUser = {
  id?: number;
  name?: string;
  age?: number;
};
```

工具类型的底层依赖几个能力：

1. `keyof` 获取对象 key。
2. 映射类型遍历 key。
3. 条件类型做类型判断。
4. `infer` 做类型推断。

## 常见工具类型

### Partial

把对象所有属性变成可选。

```ts
type UpdateUser = Partial<User>;
```

常用于编辑表单、局部更新参数。

### Required

把对象所有属性变成必选。

```ts
type FullUser = Required<Partial<User>>;
```

适合把中间态数据转换为完整数据结构时使用。

### Pick

从对象类型中挑选部分属性。

```ts
type UserBase = Pick<User, "id" | "name">;
```

常用于列表项、简略信息、组件只需要部分字段的场景。

### Omit

从对象类型中排除部分属性。

```ts
type CreateUser = Omit<User, "id">;
```

常用于创建参数，因为创建时通常没有后端生成的 `id`。

### Record

描述 key 到 value 的映射。

```ts
type RoleMap = Record<string, string[]>;
```

常用于字典、枚举映射、配置表。

### ReturnType

获取函数返回值类型。

```ts
function getUser() {
  return {
    id: 1,
    name: "Tom",
  };
}

type UserResult = ReturnType<typeof getUser>;
```

适合复用函数推导出的返回结构，避免重复声明。

## 手写工具类型

理解工具类型时，可以手写几个简化版。

### MyPartial

```ts
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};
```

这里 `[K in keyof T]` 表示遍历对象的所有 key，`?` 表示把属性改成可选。

### MyPick

```ts
type MyPick<T, K extends keyof T> = {
  [P in K]: T[P];
};
```

这里 `K extends keyof T` 保证只能挑选对象上真实存在的 key。

### MyReturnType

```ts
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
```

这里 `infer R` 表示在条件类型中推断函数返回值类型。

## 面试回答

工具类型可以理解为类型层面的工具函数，它们基于已有类型生成新类型。

比如 `Partial<T>` 可以把对象所有属性变成可选，适合编辑表单或局部更新。`Pick<T, K>` 用来挑选部分字段，`Omit<T, K>` 用来排除部分字段，`Record<K, T>` 用来描述字典结构，`ReturnType<T>` 可以拿到函数返回值类型。

它们背后的核心是 `keyof`、映射类型、条件类型和 `infer`。比如 `Partial` 的实现就是遍历 `keyof T`，然后把每个属性加上可选标记；`ReturnType` 则是通过条件类型判断一个类型是不是函数，并用 `infer` 推断返回值。

实际项目里，我会用工具类型减少重复声明，让接口参数、表单状态、组件 props 之间保持类型一致。

## 高频追问

### Pick 和 Omit 有什么区别？

`Pick` 是从对象类型中挑选字段，`Omit` 是从对象类型中排除字段。

如果只需要少数字段，用 `Pick` 更直观；如果只排除少数字段，用 `Omit` 更方便。

### Partial 有什么风险？

`Partial` 会把所有属性都变成可选，容易让类型变得过宽。

如果只想让部分字段可选，不应该直接对整个大对象使用 `Partial`，可以配合 `Pick` 或单独定义更精确的类型。

### Record 适合什么场景？

`Record` 适合描述 key-value 映射，比如状态文案映射、权限配置、枚举对应组件。

```ts
type Status = "success" | "error";

const statusText: Record<Status, string> = {
  success: "成功",
  error: "失败",
};
```

这样可以保证每个状态都有对应配置。

### infer 是什么？

`infer` 是条件类型里的类型推断关键字。

它可以从一个复杂类型里推断出某个局部类型，比如从函数类型里推断返回值，从 Promise 里推断成功值类型。
