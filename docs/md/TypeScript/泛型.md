# TypeScript 泛型

## 面试定位

这道题主要考察你是否理解泛型解决了什么问题。不要只说“泛型就是类型参数”，还要说明它如何在复用代码的同时保留类型关系。

泛型是 TypeScript 面试高频点，通常会和 `any`、类型推导、泛型约束、接口泛型、工具类型一起追问。

## 核心原理

泛型可以理解为类型层面的参数。函数、接口、类在定义时不固定具体类型，而是在使用时再确定类型。

它解决的是两个问题：

1. 代码要复用。
2. 类型关系不能丢。

如果用 `any`，虽然代码可以复用，但类型信息会丢失。

```ts
function identity(value: any): any {
  return value;
}
```

使用泛型后，参数类型和返回值类型之间的关系会被保留下来。

```ts
function identity<T>(value: T): T {
  return value;
}

const result = identity("hello");
```

这里 `result` 会被推导为 `string`。

## 泛型约束

泛型默认可以是任意类型。如果函数内部需要访问某些属性，就要通过 `extends` 添加约束。

```ts
function getLength<T extends { length: number }>(value: T): number {
  return value.length;
}

getLength("hello");
getLength([1, 2, 3]);
```

这表示传入的类型不固定，但必须具备 `length` 属性。

## keyof 与泛型

`keyof` 经常和泛型一起使用，用来约束对象属性名。

```ts
function getValue<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = {
  id: 1,
  name: "Tom",
};

const name = getValue(user, "name");
```

这里 `K extends keyof T` 表示 `key` 必须是 `obj` 上真实存在的属性名，返回值类型也能根据属性名精确推导。

## 项目使用场景

### 接口返回结构

```ts
type ApiResponse<T> = {
  code: number;
  message: string;
  data: T;
};

type UserResponse = ApiResponse<{
  id: number;
  name: string;
}>;
```

这样可以复用接口外层结构，同时保留不同接口的 `data` 类型。

### 通用组件

```ts
type SelectOption<T> = {
  label: string;
  value: T;
};
```

泛型可以让组件支持不同的 `value` 类型，例如字符串、数字、对象，同时不牺牲类型安全。

## 面试回答

泛型的核心是把类型作为参数传入，让代码在保持复用的同时保留类型关系。

比如一个函数接收什么类型，就返回什么类型。如果用 `any`，虽然可以接收任意值，但返回值也会变成 `any`，后续类型检查就失效了。泛型则可以让编译器在调用时推导出具体类型，并把这个类型贯穿到参数、返回值、对象属性等位置。

实际项目里，我会在接口响应结构、通用组件、工具函数里使用泛型。比如 `ApiResponse<T>` 可以统一接口外层结构，同时让不同接口的 `data` 保持准确类型。对于需要访问属性的泛型，我会用 `extends` 做约束，比如要求参数必须有 `length`，或者配合 `keyof` 限制 key 必须来自目标对象。

所以泛型不是为了写得复杂，而是为了让复用代码仍然有准确的类型约束。

## 高频追问

### 泛型和 any 有什么区别？

`any` 是放弃类型检查，泛型是保留类型关系。

`any` 能让代码通过编译，但会丢失输入和输出之间的类型联系。泛型可以根据调用时传入的类型推导结果，让后续代码继续受到类型保护。

### 泛型约束是什么？

泛型约束是用 `extends` 限制泛型必须满足某种结构。

比如 `T extends { length: number }` 表示传入类型可以不同，但必须有 `length` 属性。

### keyof 在泛型里有什么作用？

`keyof` 可以拿到对象类型的所有 key，配合泛型可以约束属性访问。

典型写法是 `K extends keyof T`，它能保证传入的 key 一定存在于对象上，并且返回值类型可以精确到 `T[K]`。

### 泛型会增加运行时代码吗？

不会。泛型是类型系统能力，编译后会被擦除，不会直接增加运行时代码。

这也是为什么 TypeScript 不能完全替代运行时校验。
